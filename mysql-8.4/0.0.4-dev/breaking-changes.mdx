This page lists breaking changes on the 0.0.4-dev branch. Each entry states the user-facing impact and the action required.

## Rebuild required: VEF Protocol 3

`VEF_PROTOCOL_2` is deprecated. The server rejects extensions that declare `VEF_PROTOCOL_2` at install time.

The stable protocol as of v0.0.4 is `VEF_PROTOCOL_3`. The current SDK declares `VEF_PROTOCOL_4`, which is under development. Rebuilding your extension against the current SDK is enough — no code changes are required unless your extension reads the protocol constant directly.

## `.param()` for zero-arity VDFs renamed to `.no_params()`

The zero-arity overload of `FuncBuilder::param()` is now `FuncBuilder::no_params()`. Calls to `.param()` with no arguments will not compile.

**Action:** In `make_func<...>(...)` chains that declare a zero-argument VDF, replace `.param()` with `.no_params()`.

```cpp
// Before
make_func<&my_impl>("MY_FUNC")
  .returns(INT)
  .param()
  .build()

// After
make_func<&my_impl>("MY_FUNC")
  .returns(INT)
  .no_params()
  .build()
```

VDFs that take typed arguments are unaffected — `.param(TYPE)` overloads with arguments are unchanged.

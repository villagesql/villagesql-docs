title: "Your First Extension"
description: "Build a minimal working VillageSQL extension in under 30 lines of C++."
---

This page gets you to a working extension as fast as possible. It covers the minimum required to register a single SQL function and install it. For the full build workflow, see [Creating Extensions](/mysql-8.4/0.0.4-dev/create).

## Prerequisites

- A built VillageSQL server. Follow [Build from Source](/mysql-8.4/0.0.4-dev/source) first.
- CMake 3.16+, a C++17 compiler, and Git.

## Step 1: Get the Template

Fork or clone the extension template:

```bash
git clone https://github.com/villagesql/vsql-extension-template.git my_extension
cd my_extension
```

## Step 2: Write Your Function

Open `src/extension.cc` and replace its contents with:

```cpp
#include <villagesql/vsql.h>

using namespace vsql;

void double_it_impl(IntArg n, IntResult out) {
    if (n.is_null()) { out.set_null(); return; }
    out.set(n.value() * 2);
}

VEF_GENERATE_ENTRY_POINTS(
    make_extension()
        .func(make_func<&double_it_impl>("double_it")
            .returns(INT)
            .param(INT)
            .build())
)
```

## Step 3: Update the Manifest

Edit `manifest.json`:

```json
{
  "name": "my_extension",
  "version": "1.0.0",
  "description": "Returns double the input integer"
}
```

## Step 4: Build

```bash
mkdir build && cd build
cmake .. -DVillageSQL_BUILD_DIR=/path/to/villagesql/build
make
```

This produces `my_extension.veb`.

## Step 5: Install and Test

Copy the `.veb` file to your server's `veb_dir`:

```bash
# Find veb_dir
mysql -u root -e "SHOW VARIABLES LIKE 'veb_dir';"

cp my_extension.veb /path/to/veb_dir/
```

Install and call the function:

```sql
INSTALL EXTENSION my_extension;

SELECT double_it(21);
-- 42

SELECT double_it(NULL);
-- NULL

UNINSTALL EXTENSION my_extension;
```

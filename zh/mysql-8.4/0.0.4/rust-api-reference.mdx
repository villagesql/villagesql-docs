## InValue

`InValue` 是服务器为每个函数参数传递的枚举。您的函数接收 `args: &[InValue]`，必须在使用其值前检查每个参数。

```rust
pub enum InValue<'a> {
    String(&'a str),
    Real(f64),
    Int(i64),
    Null,
    Custom(&'a [u8]),
}
```

| 枚举变体 | Rust 类型 | 对应 SQL 类型 |
|---|---|---|
| `String(&str)` | UTF-8 字符串切片 | `STRING` / `VARCHAR` / `TEXT` |
| `Real(f64)` | 64 位浮点数 | `REAL` / `DOUBLE` / `FLOAT` |
| `Int(i64)` | 64 位有符号整数 | `INT` / `BIGINT` / `TINYINT` |
| `Null` | — | 任意类型的 SQL `NULL` |
| `Custom(&[u8])` | 原始二进制字节 | 通过 `custom_type!` 注册的任意自定义类型 |

始终显式匹配 `Null`。调用 `.unwrap()` 或仅匹配值变体的模式匹配是错误的 —— SQL `NULL` 是正常输入，而非错误。

## VdfReturn

`VdfReturn` 是您的函数返回给服务器的类型。通过关联函数构造：

| 构造函数 | SQL 效果 |
|---|---|
| `VdfReturn::null()` | 为当前行返回 SQL `NULL` |
| `VdfReturn::string(s)` | 返回 `String` 值；`s` 是 `impl Into<String>` |
| `VdfReturn::real(v)` | 返回 `f64` 值 |
| `VdfReturn::int(v)` | 返回 `i64` 值 |
| `VdfReturn::binary(bytes)` | 返回自定义类型列的二进制字节；`bytes` 是 `Vec<u8>` |
| `VdfReturn::warning(msg)` | 为当前行返回 `NULL`，添加 SQL 警告，继续执行 |
| `VdfReturn::error(msg)` | 以致命错误终止语句 |

**警告与错误的区别：**

在用户输入验证失败且继续处理结果集合理时，使用 `warning`。在严格模式下，MySQL 会将 `INSERT` 和 `UPDATE` 中的警告提升为错误。在继续执行不安全（如存储数据损坏、内部不变量违反）时，使用 `error`。致命错误会终止整个语句。

```rust
fn validate_impl(args: &[InValue]) -> VdfReturn {
    match args.first() {
        Some(InValue::Int(n)) if *n >= 0 => VdfReturn::int(*n),
        Some(InValue::Int(_)) => VdfReturn::warning("value must be non-negative"),
        Some(InValue::Null) | None => VdfReturn::null(),
        _ => VdfReturn::error("validate: expected an INT argument"),
    }
}
```

## extension! 宏

`extension!` 生成服务器加载 VEB 文件时调用的 VEF 入口点。它必须在 crate 中恰好出现一次。

```rust
vsql::extension! {
    funcs: [
        // One or more vsql::func!(...) declarations
    ],
    types: [
        // One or more vsql::custom_type!(...) declarations
    ]
}
```

两部分均可选。纯函数扩展省略 `types:`；仅类型扩展省略 `funcs:`。空的 `extension!` 块（无 funcs，无 types）有效，但生成的扩展无实际功能。

## func! 宏

`func!` 声明一个 SQL 可调用函数。位置参数：

```rust
vsql::func!(rust_fn, "sql_name", [param_types] -> return_type)
vsql::func!(rust_fn, "sql_name", [param_types] -> return_type, deterministic: true)
```

| 参数 | 描述 |
|---|---|
| `rust_fn` | 实现 VDF 的 Rust 函数。签名：`fn(&[InValue]) -> VdfReturn`。 |
| `"sql_name"` | SQL 函数名称（字符串字面量）。这是用户在 SQL 中调用的名称。 |
| `[param_types]` | 逗号分隔的 `vsql::Type::*` 或 `vsql::custom!("name")` 值列表。零参数函数使用 `[]`。 |
| `return_type` | `vsql::Type::*` 或 `vsql::custom!("name")`。 |
| `deterministic: true` | 可选。声明函数为确定性 —— 相同输入始终产生相同输出，无副作用。优化器可缓存相同输入的结果。仅在确实为确定性时设置。 |

**func! 中使用的类型常量：**

| `vsql::Type::*` | SQL 类型 |
|---|---|
| `vsql::Type::String` | `STRING` |
| `vsql::Type::Real` | `REAL` |
| `vsql::Type::Int` | `INT` |

## custom_type! 宏

`custom_type!` 注册新列类型。`type_name`、`persisted_length`、`max_decode_buffer_length`、`encode`、`decode` 和 `compare` 为必填项。`hash` 和 `default` 为可选但推荐。

```rust
vsql::custom_type!(
    type_name: "sql_type_name",
    persisted_length: N,
    max_decode_buffer_length: M,
    encode: encode_fn,
    decode: decode_fn,
    compare: compare_fn,
    hash: hash_fn,
    default: "default_string",
)
```

| 字段 | 类型 | 描述 |
|---|---|---|
| `type_name` | `&str` 字面量 | 类型的 SQL 名称。SQL 中不区分大小写。必须在所有已安装扩展中唯一。 |
| `persisted_length` | `usize` | 磁盘存储的固定字节长度。所有编码值必须产生完全相同的字节数。 |
| `max_decode_buffer_length` | `usize` | 解码后字符串的最大字节长度。用于在调用 `decode` 前分配输出缓冲区。 |
| `encode` | `fn(&str) -> Result<Vec<u8>, String>` | 在 `INSERT` 时调用。将 SQL 字符串字面量转换为二进制。返回 `Err(msg)` 拒绝输入。 |
| `decode` | `fn(&[u8]) -> Result<String, String>` | 用于显示值。将二进制转换回字符串。 |
| `compare` | `fn(&[u8], &[u8]) -> std::cmp::Ordering` | 用于 `ORDER BY`、`MIN`、`MAX`。返回 `Less`、`Equal` 或 `Greater`。 |
| `hash` | `fn(&[u8]) -> usize` | 可选。用于 `COUNT(DISTINCT)` 和集合操作。比较 `Equal` 的值必须哈希到相同值。推荐用于索引列。 |
| `default` | `&str` 字面量 | 可选。服务器在类型初始化时编码的合法字符串，用于验证回调是否正常工作。必须编码为恰好 `persisted_length` 字节。 |

`default` 字段并非列默认值 —— 它是启动探测。服务器在加载扩展时调用 `encode(default)` 以验证回调是否正常工作。如果 `encode` 对 `default` 返回 `Err`，扩展将加载失败。

## custom! 宏

`vsql::custom!("type_name")` 在 `func!` 声明中通过名称引用自定义类型：

```rust
vsql::func!(
    my_fn,
    "my_sql_func",
    [vsql::custom!("mytype")] -> vsql::custom!("mytype"),
    deterministic: true
)
```

它可出现在 `vsql::Type::*` 会出现在参数列表或返回类型位置的任何地方。字符串必须与对应 `custom_type!` 中声明的 `type_name` 匹配。

## manifest.json 字段

每个扩展都需要在 `Cargo.toml` 旁提供 `manifest.json`：

```json
{
  "name": "vsql_my_extension",
  "version": "0.1.0",
  "description": "Brief description of what the extension does",
  "author": "Your Name",
  "license": "GPL-2.0"
}
```

| 字段 | 必填 | 格式 | 描述 |
|---|---|---|---|
| `name` | 是 | 小写字母、数字、`_`、`-` | 扩展标识符。必须与 `INSTALL EXTENSION` 名称匹配。建议使用下划线 —— 短横线在 SQL 中需要反引号转义。 |
| `version` | 是 | MAJOR.MINOR.PATCH | 语义化版本。 |
| `description` | 否 | 字符串 | 在 `INFORMATION_SCHEMA.EXTENSIONS` 中显示。 |
| `author` | 否 | 字符串 | 作者名称或组织。 |
| `license` | 否 | 字符串 | 许可证标识符。开源扩展推荐使用 `GPL-2.0`。 |

`name` 验证规则：必须以字母开头，以字母或数字结尾，最大 64 个字符。无效的 manifest 会导致 `INSTALL EXTENSION` 失败。

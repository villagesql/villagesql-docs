## 概述

从源代码构建 VillageSQL 可以获得最新功能，并允许您为特定环境自定义构建。

## 先决条件

开始之前，请确保已安装以下软件：

- **Git** - 用于克隆仓库
- **CMake** 3.16 或更高版本 - 构建系统生成器
- **C++ 编译器** - GCC 8+、Clang 8+ 或 MSVC 2019+
- **构建工具** - make、ninja 或等效工具
- **开发库** - OpenSSL、ncurses、pkg-config、bison 以及其他 MySQL 依赖项

### 安装依赖项

**Ubuntu/Debian:**
```bash
sudo apt install cmake libssl-dev libncurses5-dev pkg-config bison \
                 libtirpc-dev rpcsvc-proto build-essential zlib1g-dev
```

**macOS（使用 Homebrew）：**

首先，如果尚未安装 Homebrew，请先安装 Homebrew：
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

然后安装依赖项：
```bash
brew install cmake openssl pkgconf bison libtirpc rpcsvc-proto
```

## 步骤 1：克隆仓库

从 GitHub 克隆 VillageSQL Server 仓库。克隆到您的主目录，以便下面的 CMake 步骤无需修改即可正常工作：

```bash
cd $HOME
git clone --depth 1 https://github.com/villagesql/villagesql-server.git
cd villagesql-server
```

<Note>
  由于包含 MySQL 代码库，仓库大小达数 GB。
</Note>

## 步骤 2：使用 CMake 配置

在仓库外部创建构建目录并配置项目：

**Linux：**
```bash
# Create build directory (outside the repo)
mkdir -p $HOME/build/villagesql
cd $HOME/build/villagesql

# Configure with CMake
cmake $HOME/villagesql-server -DWITH_DEBUG=1 -DCMAKE_INSTALL_PREFIX=$HOME/mysql
```

**macOS：**
```bash
# Create build directory (outside the repo)
mkdir -p ~/build/villagesql
cd ~/build/villagesql

# Configure with CMake
cmake ~/villagesql-server -DWITH_DEBUG=1 -DCMAKE_INSTALL_PREFIX=~/mysql -DWITH_SSL=system
```

<Note>
  **Linux 用户：** 使用 `$HOME` 表示绝对路径。**macOS 用户：** 使用 `~`（波浪号）。如果克隆位置不同，请替换仓库路径。
</Note>

### CMake 选项说明

- `<path-to-repo>` - 克隆的 VillageSQL 仓库路径（Linux 使用 `$HOME`，macOS 使用 `~`）
- `-DWITH_DEBUG=1` - 启用调试符号（开发推荐）
- `-DCMAKE_INSTALL_PREFIX=~/mysql` - 设置安装目录
- `-DWITH_SSL=system` - 使用系统 OpenSSL 库（macOS 必需）

### 额外 CMake 选项

**无调试符号的生产环境构建：**

Linux：
```bash
cmake $HOME/villagesql-server -DCMAKE_INSTALL_PREFIX=/usr/local/mysql
```

macOS：
```bash
cmake ~/villagesql-server -DCMAKE_INSTALL_PREFIX=/usr/local/mysql -DWITH_SSL=system
```

**带自定义编译注释：**

Linux：
```bash
cmake $HOME/villagesql-server -DWITH_DEBUG=1 \
      -DCMAKE_INSTALL_PREFIX=$HOME/mysql \
      -DCOMPILATION_COMMENT="VillageSQL Version of MySQL"
```

macOS：
```bash
cmake ~/villagesql-server -DWITH_DEBUG=1 \
      -DCMAKE_INSTALL_PREFIX=~/mysql \
      -DCOMPILATION_COMMENT="VillageSQL Version of MySQL"
```

**启用更严格警告的开发者模式：**

Linux：
```bash
cmake $HOME/villagesql-server -DMYSQL_MAINTAINER_MODE=ON -DWITH_DEBUG=1
```

macOS：
```bash
cmake ~/villagesql-server -DMYSQL_MAINTAINER_MODE=ON -DWITH_DEBUG=1
```

<Note>
  如需重新配置，请先清除 CMake 缓存（在构建目录内执行）：
  ```bash
  rm CMakeCache.txt
  ```
</Note>

## 步骤 3：编译代码

使用 make 进行并行编译构建 VillageSQL。在构建目录内执行：

**仅构建服务器（开发推荐）：**
```bash
make -j10 mysqld
```

**构建全部内容：**
```bash
make -j10
```

<Tip>
  根据 CPU 核心数调整并行度（`-j10`）。建议将总核心数减去 2-4 以保持系统响应。例如，在 12 核机器上使用 `-j10`。
</Tip>

完成编译后，验证服务器二进制文件已成功构建：

**Linux：**
```bash
ls $HOME/build/villagesql/bin/mysqld
```

**macOS：**
```bash
ls ~/build/villagesql/bin/mysqld
```

## 步骤 4：初始化数据库

首次启动服务器前，初始化数据目录：

**Linux：**

生产环境（使用生成的密码 - 推荐）：
```bash
mkdir -p $HOME/mysql-data/data
$HOME/build/villagesql/bin/mysqld --initialize --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

开发环境（无密码 - 可选）：
```bash
mkdir -p $HOME/mysql-data/data
$HOME/build/villagesql/bin/mysqld --initialize-insecure --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

**以 root 身份运行（Docker 或 sudo）：**

若以 root 身份运行（例如在 Docker 中），MySQL 需要 `--user=root` 标志：
```bash
# Initialize as root
$HOME/build/villagesql/bin/mysqld --user=root --initialize-insecure --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

**macOS：**

生产环境（使用生成的密码 - 推荐）：
```bash
mkdir -p ~/mysql-data/data
~/build/villagesql/bin/mysqld --initialize --datadir=~/mysql-data/data --basedir=~/build/villagesql
```

开发环境（无密码 - 可选）：
```bash
mkdir -p ~/mysql-data/data
~/build/villagesql/bin/mysqld --initialize-insecure --datadir=~/mysql-data/data --basedir=~/build/villagesql
```

<Note>
  生产环境使用 `--initialize`（带密码）。仅在本地开发和测试时使用 `--initialize-insecure`（无密码）。使用 `--initialize` 时，临时密码将生成并打印到控制台：`A temporary password is generated for root@localhost: <password>`
</Note>

通过检查系统数据库是否已创建来验证初始化是否成功：

**Linux：**
```bash
ls $HOME/mysql-data/data/mysql
```

**macOS：**
```bash
ls ~/mysql-data/data/mysql
```

## 步骤 5：启动服务器

启动 VillageSQL 服务器：

**Linux：**
```bash
$HOME/build/villagesql/bin/mysqld --gdb --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

**以 root 身份运行（Docker 或 sudo）：**
```bash
$HOME/build/villagesql/bin/mysqld --user=root --gdb --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

**macOS：**
```bash
~/build/villagesql/bin/mysqld --gdb --datadir=~/mysql-data/data --basedir=~/build/villagesql
```

<Tip>
  `--gdb` 标志会安装 `SIGINT` 处理器，使 Ctrl-C 能干净地停止服务器——在终端中交互运行时非常有用。如需在后台运行，可在 `mysqld` 命令中添加 `--daemonize`。
</Tip>

## 步骤 6：使用 MySQL 客户端连接

打开新终端，使用 MySQL 客户端连接服务器：

**Linux：**

若使用 `--initialize-insecure`（无密码）：
```bash
$HOME/build/villagesql/bin/mysql -u root
```

若使用 `--initialize`（带生成的密码）：
```bash
$HOME/build/villagesql/bin/mysql -u root -p
# Enter the temporary password printed during initialization
```

**macOS：**

若使用 `--initialize-insecure`（无密码）：
```bash
~/build/villagesql/bin/mysql -u root
```

若使用 `--initialize`（带生成的密码）：
```bash
~/build/villagesql/bin/mysql -u root -p
# Enter the temporary password printed during initialization
```

您应看到 MySQL 提示符：

```
Welcome to the VillageSQL Server for MySQL monitor.
Type 'help;' or '\h' for help. Type '\c' to clear the current input statement.

mysql>
```

### 验证安装

检查是否正在运行 VillageSQL：

```sql
SELECT VERSION();
```

开发构建会在版本字符串中包含 git 提交哈希：
```
8.4.9-villagesql-0.0.4
```

## 步骤 7：设置用户和数据库

### 更改 root 密码

若使用了 `--initialize`，请更改临时密码：

```sql
SET PASSWORD = 'your-secure-password';
```

### 创建开发用户

为日常开发创建非 root 用户：

```sql
-- Create user
CREATE USER developer IDENTIFIED BY 'dev-password';

-- Grant all privileges
GRANT ALL PRIVILEGES ON *.* TO developer;
```

退出并以新用户重新连接：

**Linux：**
```bash
# Ctrl-D to exit
$HOME/build/villagesql/bin/mysql -u developer -p
```

**macOS：**
```bash
# Ctrl-D to exit
~/build/villagesql/bin/mysql -u developer -p
```

### 创建数据库

```sql
CREATE DATABASE my_database;
USE my_database;
```

<Tip>
  连接到特定数据库：`mysql -u developer -p -D my_database`
</Tip>

如需使用 GDB 调试、运行测试或贡献服务器代码，请参阅 [服务器开发指南](/mysql-8.4/0.0.4/server-development)。

## 故障排除

### 构建因缺少依赖项失败

为您的平台安装所需的开发包。检查错误消息以确定缺少的特定库。

### 服务器无法启动

- 验证数据目录是否已初始化：`ls ~/mysql-data/data/`
- 检查是否有其他 MySQL/VillageSQL 实例占用端口 3306
- 查看 `~/mysql-data/data/*.err` 中的错误日志

### 扩展安装失败

- 确保扩展库（`.so` 或 `.dll`）存在于构建输出中
- 检查 VillageSQL 是否具有加载扩展的必要权限
- 验证扩展名称和 `.veb` 文件名是否正确

---
title: "扩展或插件与组件"
description: "何时构建 VEF 扩展而非使用 MySQL 原生插件或组件接口。"
---

VillageSQL 是 MySQL 的跟踪分支，可作为即插即用替代品。MySQL 插件和组件接口在 VillageSQL 中的工作方式与在 MySQL 中完全相同。VillageSQL 提供了 VillageSQL 扩展框架（VEF）。对于添加新功能——自定义 SQL 函数、自定义类型、集成等——VEF 扩展是正确的模型。

## 何处构建

如果您正在构建新功能，请在您自己的仓库中构建 VEF 扩展。插件和组件也能工作，但 VEF 是正确的模型——参见 [为何使用扩展而非插件和组件](#why-extensions-instead-of-plugins-and-components)。

应提交至 VillageSQL Server 仓库的贡献：

| | |
|---|---|
| VEF 本身的改进（新钩子类型、SDK 能力） | 提交至 [server 仓库](https://github.com/villagesql/villagesql-server) 的 PR |
| 服务器错误修复 | 在 [server 仓库](https://github.com/villagesql/villagesql-server) 中提交 Issue 或 PR |

## 为何使用扩展而非插件和组件

MySQL 插件和组件在 VillageSQL 中可以正常工作。VEF 扩展提供额外优势：

- **自定义类型** —— 定义新列类型，插件和 UDF 无法实现
- **打包方式** —— 以 `.veb` 文件分发，通过 `INSTALL EXTENSION` 安装
- **无需重启** —— 扩展可在运行时加载和卸载
- **版本控制** —— 在扩展清单中声明，由服务器强制执行

## 开始新扩展

克隆或 fork [vsql-extension-template](https://github.com/villagesql/vsql-extension-template)。它包含一个可工作的 Hello World 扩展、清单格式和完整的 CMake 构建设置。

拥有仓库后，[创建扩展](/mysql-8.4/0.0.4/create) 会逐步介绍 VEF SDK。

## 若 VEF 不支持所需功能

VEF 仍在发展中。若您需要尚未支持的钩子或能力：

- **提交 Issue** 至 [server 仓库](https://github.com/villagesql/villagesql-server/issues)，描述所需功能
- **为现有 Issue 点赞**（若请求已存在）
- **提交 PR** 为 VEF 添加钩子 —— 框架改进欢迎提交至 server 仓库

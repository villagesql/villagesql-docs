</Warning>

## 概述

VillageSQL 的扩展框架允许您向数据库服务器添加自定义功能。使用 VEF SDK 和扩展模板构建自定义扩展。

本指南涵盖构建和安装扩展的端到端步骤。如需深入了解编写 VDF 实现（包括类型包装器、聚合函数、系统变量、参数化类型）——请参阅 [开发指南](/mysql-8.4/0.0.4/development)。

<Note>
如果您更喜欢 Rust，请参阅 [使用 Rust 构建扩展](/mysql-8.4/0.0.4/rust-sdk)。
</Note>

## 什么是 VillageSQL 扩展？

VillageSQL 扩展打包为 **VEB 文件**（VillageSQL 扩展捆绑包），包含：
- **清单** - 扩展的元数据（名称、版本、描述）
- **共享库** - 实现功能的编译 C++ 代码
- **可选元数据** - 额外资源或配置

扩展使用 **VEF SDK**（VillageSQL 扩展框架）构建，提供：
- 用于定义类型和函数的 C++ API
- 无需 SQL 脚本的自动注册
- 类型安全的函数包装器
- 扩展定义的构建器模式

<Note>
**VDF 与传统 UDF 的区别：** 通过 VEF SDK 注册的函数称为 VDF（VillageSQL 定义函数）。VillageSQL 也支持通过 `CREATE FUNCTION ... SONAME` 注册的传统 MySQL UDF，但建议新扩展使用 VEF SDK 方案。
</Note>

### 在 SQL 中调用 VDF

VDF 可以使用或不使用扩展前缀调用：

```sql
-- Unqualified (preferred for cleaner code)
SELECT complex_abs(impedance) FROM signals;

-- Qualified with extension name (explicit)
SELECT vsql_complex.complex_abs(impedance) FROM signals;
```

**函数解析顺序：**

调用未限定的函数时，VillageSQL 按以下顺序解析：
1. **系统函数**（内置 MySQL 函数如 `NOW()`、`CONCAT()`）
2. **UDFs**（传统 MySQL 用户定义函数）
3. **VDFs**（扩展函数）——仅当恰好存在一个同名函数时
4. **存储函数**（通过 `CREATE FUNCTION` 创建）

**何时使用限定名称：**
- 当多个扩展提供同名函数时使用 `extension.function_name`
- 无歧义时使用未限定名称以保持代码简洁
- 仅当只有一个扩展提供该函数名时无需限定

扩展可添加：
- **自定义函数（VDFs）** - 具有自动类型检查和验证的 SQL 函数
- **自定义数据类型** - 新列类型如 COMPLEX、UUID 或 VECTOR，支持 ORDER BY 和索引
- **类型操作** - 自定义类型的编码、解码、比较和哈希函数

## 先决条件

开始前，需从源码构建 VillageSQL —— 扩展需链接到服务器的 SDK 头文件和构建树。请先遵循 [从源码构建](/mysql-8.4/0.0.4/source) 指南。

您还需要：
- **Git** - 用于克隆和版本控制
- **CMake** 3.18 或更高版本 - 构建系统
- **C++ 编译器** - GCC 8+、Clang 8+ 或 MSVC 2019+（支持 C++17）
- **基础 C++ 知识** - 理解 C++ 和函数指针

<Tip>
  **使用 AI 代理构建？** [`vsql-extension-builder`](https://github.com/villagesql/villagesql-skills) 技能可自动化此工作流（从脚手架到测试）——使用 Claude Code、Gemini 或其他支持的代理。安装方式：

  ```bash
  curl -sSL https://villagesql.com/skills | bash
  ```
</Tip>

## 步骤 1：获取扩展模板

您可通过两种方式开始使用扩展模板：

### 选项 A：使用 VillageSQL 源码中的模板

如果您有 VillageSQL 源码，模板已包含：

```bash
cd /path/to/villagesql-source
cp -r villagesql/sdk/template my-extension
cd my-extension
```

### 选项 B：从 GitHub 分叉

1. 访问 GitHub 上的模板仓库：
   ```
   https://github.com/villagesql/vsql-extension-template
   ```

2. 点击 **"Fork"** 按钮创建您自己的副本

3. 本地克隆您的分叉：
   ```bash
   git clone https://github.com/YOUR_USERNAME/vsql-extension-template.git
   cd vsql-extension-template
   ```

<Tip>
  或使用 GitHub 上的 "Use this template" 按钮，无需分叉历史即可创建基于模板的新仓库。
</Tip>

## 步骤 2：更新清单

编辑 `manifest.json` 定义扩展元数据：

```json
{
  "$schema": "https://raw.githubusercontent.com/villagesql/villagesql-docs/main/mysql-8.4/0.0.4/manifest.json.schema.json",
  "name": "my_awesome_extension",
  "version": "1.0.0",
  "description": "My custom VillageSQL extension that does amazing things",
  "author": "Your Name",
  "license": "GPL-2.0"
}
```

`$schema` 字段为可选，但可启用 IDE 自动补全和清单字段的内联验证。

### manifest.json Schema

| 字段 | 必填 | 格式 | 描述 |
|-------|-------|--------|-----|
| `name` | ✅ 是 | 字母、数字、`_`、`-` | 唯一标识符。必须匹配 `INSTALL EXTENSION` 名称。 |
| `version` | ✅ 是 | MAJOR.MINOR.PATCH | 语义化版本字符串 |
| `description` | 否 | 字符串 | 功能简要说明 |
| `author` | 否 | 字符串 | 作者名称或组织 |
| `license` | 否 | 字符串 | 许可证标识符（推荐 GPL-2.0） |

**验证规则：**
- `name`：必须以字母开头并以字母或数字结尾。可包含小写字母、数字、下划线和连字符。最大 64 个字符。**使用下划线**——连字符在 SQL 中需用反引号引用。
- `version`：必须符合语义化版本规范（如 1.0.0、0.2.1）
- 无效清单会导致 `INSTALL EXTENSION` 失败

**示例：**
```sql
-- manifest.json has "name": "my_awesome_extension"
INSTALL EXTENSION my_awesome_extension;  -- ✅ Correct: no quoting needed
INSTALL EXTENSION `my-awesome-extension`;  -- ⚠️ Works, but requires backtick quoting
```

有关 SQL、文件名和仓库名称的完整命名约定，请参阅 [扩展命名约定](/mysql-8.4/0.0.4/install#extension-naming-conventions)。

## 步骤 3：使用 VEF SDK 实现扩展

VEF SDK 提供 C++ API，使用流畅构建器模式定义扩展：
- 带编译时检查的类型安全函数定义
- 自动参数验证和类型转换
- 支持自定义类型（含比较/哈希函数，支持 ORDER BY 和索引）

### 包含 VillageSQL 头文件

创建主扩展文件（如 `src/extension.cc`）并包含 VEF SDK：

```cpp
#include <villagesql/vsql.h>

// Your implementation code here
```

`<villagesql/vsql.h>` 头文件引入类型构建器、函数构建器和扩展构建器，并将常用符号重导出到 `vsql` 命名空间。

### 定义您的扩展

使用 `VEF_GENERATE_ENTRY_POINTS()` 宏定义扩展：

```cpp
VEF_GENERATE_ENTRY_POINTS(
  make_extension()
    .func(make_func<&my_reverse_impl>("my_reverse")
      .returns(STRING)
      .param(STRING)
      .build())
    .func(make_func<&count_vowels_impl>("count_vowels")
      .returns(INT)
      .param(STRING)
      .build())
);
```

**函数构建器方法：**
- `make_func<&impl>("name")` - 使用实现指针创建函数
- `.returns(type)` - 设置返回类型（STRING、INT、REAL 或自定义类型名）
- `.param(type)` - 添加参数（最多 8 个参数）
- `.buffer_size(size_t)` - 为 STRING/CUSTOM 返回请求特定输出缓冲区大小
- `.deterministic(bool = true)` - 声明该函数对相同输入始终返回相同输出且无副作用。默认为非确定性。
- `.prerun<func>()` - 设置每语句初始化函数（可选）
- `.postrun<func>()` - 设置每语句清理函数（可选）
- `.build()` - 完成函数注册

<Note>
**参数限制：** 函数最多支持 8 个参数（由 `kMaxParams` 定义）。如需更多参数，考虑使用结构化类型或多个函数。
</Note>

### 带自定义类型参数和返回值的 VDF

VDF 可通过构建器中的 `.param(TYPE_NAME)` 和 `.returns(TYPE_NAME)` 接收和返回自定义类型值。实现使用 `CustomArg` 作为输入和 `CustomResult` 作为输出——与类型操作使用的包装器相同：

```cpp
void complex_conjugate_impl(CustomArg in, CustomResult out) {
    if (in.is_null()) { out.set_null(); return; }
    auto src = in.value();    // Span<const unsigned char> — raw binary
    auto dst = out.buffer();  // Span<unsigned char>
    // ... read src, write result to dst ...
    out.set_length(src.size());
}
```

使用 `.param(COMPLEX)` 和 `.returns(COMPLEX)` 注册：

```cpp
.func(make_func<&complex_conjugate_impl>("complex_conjugate")
          .returns(COMPLEX)
          .param(COMPLEX)
          .build())
```

有关完整 `CustomArg`/`CustomResult` API（含参数化类型的 `CustomArgWith<P>` 和 `CustomResultWith<P>`），请参阅 [开发指南](/mysql-8.4/0.0.4/development#typed-wrappers)。

### 确定性函数

默认情况下，VDF 注册为非确定性。非确定性函数在以下 SQL 上下文中被阻止：生成列、CHECK 约束和表达式默认值（`DEFAULT (expr)` 列）。如需使用这些功能，必须将函数标记为确定性。如果您的函数对相同输入始终产生相同输出且无副作用，可通过在构建器链中添加 `.deterministic()` 声明其为确定性。

优化器可能利用此信息，对每条语句仅计算一次函数值并在行间复用，而非每行调用。错误地标记非确定性函数为确定性可能导致服务器对本应产生不同输出的输入返回相同结果。仅当函数确实不依赖外部状态、随机性或时间时，才添加 `.deterministic()`。有关确定性要求的详细信息，请参阅 [MySQL UDF 特性文档](https://dev.mysql.com/doc/refman/8.4/en/udf-characteristics.html)。

**构建器签名：** `.deterministic(bool d = true)` — 无参数形式默认为 `true`。

**示例：**

```cpp
.func(make_func<&complex_add_impl>("complex_add")
          .returns(COMPLEX)
          .param(COMPLEX)
          .param(COMPLEX)
          .deterministic()
          .build())
```

由于 `complex_add` 被声明为确定性，可将其用于生成列定义：

```sql
-- Deterministic VDFs can be used in generated columns
CREATE TABLE t (
    a COMPLEX,
    b COMPLEX,
    result COMPLEX GENERATED ALWAYS AS (complex_add(a, b)) STORED
);
-- STORED vs VIRTUAL follows standard MySQL generated column rules
```

### 自定义缓冲区大小

对于返回可变长度数据的函数，请求特定缓冲区大小：

```cpp
make_func<&large_result_impl>("large_result")
  .returns(STRING)
  .param(INT)
  .buffer_size(65536)  // Request 64KB buffer
  .build()
```

写入前检查可用缓冲区空间：

```cpp
void large_result_impl(StringArg input, StringResult out) {
    if (input.is_null()) { out.set_null(); return; }
    size_t needed = calculate_output_size(input.value());

    auto buf = out.buffer();
    if (needed > buf.size()) {
        out.error("Output exceeds buffer size");
        return;
    }

    // Write output into buf.data()
    out.set_length(actual_output_length);
}
```

<Note>
根据函数最大输出大小，通过 `.buffer_size()` 请求足够缓冲区大小。
</Note>

<Note>
在实现函数前，请查阅 [扩展 API 参考](/mysql-8.4/0.0.4/extension-api-reference) 了解完整 VDF 合约：空值检查、结果类型、缓冲区大小和错误处理。
</Note>

## 步骤 4：创建自定义类型

自定义类型允许您定义新列类型（如 `COMPLEX`、`UUID` 或 `VECTOR`），支持 `ORDER BY`、索引和聚合函数。如果您的扩展仅注册函数，请跳至步骤 5。

有关完整实现，请参阅 [创建自定义类型](/mysql-8.4/0.0.4/custom-types)。如您的类型需要参数（如 `VECTOR(1536)`），请参阅 [参数化类型](/mysql-8.4/0.0.4/development#parameterized-types)。

## 步骤 5：更新构建配置

编辑 `CMakeLists.txt` 以构建为 VEB 文件：

```cmake
cmake_minimum_required(VERSION 3.18)
project(my_extension)

# Find VillageSQL Extension Framework
find_package(VillageSQLExtensionFramework QUIET)

# The framework detects build flags via 4 methods (in order):
# 1. Explicit MYSQL_INCLUDE_FLAGS/MYSQL_CXXFLAGS
# 2. VillageSQL_BUILD_DIR (reads CMakeCache.txt from VillageSQL build)
# 3. VSQL_BASE_DIR (uses mysql_config)
# 4. Default - mysql_config from PATH

# Build shared library with your source files
add_library(extension SHARED
    src/extension.cc
    src/my_functions.cc
)

# Create VEB archive
VEF_CREATE_VEB(
    NAME my_extension
    LIBRARY_TARGET extension
    MANIFEST ${CMAKE_CURRENT_SOURCE_DIR}/manifest.json
)

# Install VEB to VillageSQL extensions directory
install(FILES ${VEB_OUTPUT} DESTINATION ${INSTALL_DIR})
```

**配置说明：**
- `VillageSQLExtensionFramework` 提供构建扩展的 CMake 辅助工具
- `VEF_CREATE_VEB()` 将库、清单和元数据打包为 `.veb` 归档
- 框架自动检测 MySQL/VillageSQL 构建标志
- 库目标名称通常为 `extension`（可自定义）
- VEB 名称必须匹配 `manifest.json` 名称
- 默认情况下，扩展使用稳定 ABI 头文件构建。设置 `-DVSQL_USE_DEV_ABI=ON` 以使用不稳定开发头文件

## 步骤 6：创建构建目录

创建独立的构建目录：

```bash
mkdir build
cd build
```

## 步骤 7：使用 CMake 和 Make 构建

配置并构建扩展：

```bash
# Configure the build
cmake ..

# Or, if building against VillageSQL source:
cmake .. -DVillageSQL_BUILD_DIR=/path/to/villagesql/build

# Or, to build against the unstable dev ABI headers:
cmake .. -DVSQL_USE_DEV_ABI=ON

# Build the extension
make
```

这将创建：
- 编译后的共享库（`.so` 文件）
- VEB 包（`.veb` 文件）——包含清单和库的 tar 归档

### 验证构建

检查 VEB 文件内容：

```bash
make show_veb
```

您应看到：
```
manifest.json
lib/myext.so
```

## 步骤 8：安装和测试

### 选项 A：安装到 VillageSQL 扩展目录

使用安装目标将 VEB 复制到 VillageSQL 安装目录：

```bash
make install
```

这会将 `.veb` 文件复制到通过 `VillageSQL_VEB_INSTALL_DIR` 配置的目录。

### 选项 B：手动安装

手动复制 VEB 文件：

```bash
# Find the VEF directory
mysql -u root -p -e "SHOW VARIABLES LIKE 'veb_dir';"

# Copy the VEB file
sudo cp my-awesome-extension.veb /path/to/veb_dir/
```

### 测试您的扩展

1. **连接到 VillageSQL**：
   ```bash
   mysql -u root -p
   ```

2. **安装扩展**：
   ```sql
   INSTALL EXTENSION my_awesome_extension;
   ```

3. **验证安装**：
   ```sql
   SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;
   ```

4. **测试函数**：
   ```sql
   SELECT my_reverse('Hello, World!');
   -- Output: !dlroW ,olleH

   SELECT count_vowels('VillageSQL');
   -- Output: 3
   ```

## 创建测试

添加测试文件以验证扩展正确运行：

1. **在 `mysql-test/t/` 中创建测试文件**：
   ```sql
   -- mysql-test/t/my_basic.test
   SELECT my_reverse('abc');
   SELECT my_reverse('');
   SELECT my_reverse(NULL);
   ```

2. **生成预期结果**：
   ```bash
   cd /path/to/villagesql/build/mysql-test
   perl mysql-test-run.pl --suite=/path/to/your/extension/mysql-test --record
   ```

3. **运行测试**：
   ```bash
   perl mysql-test-run.pl --suite=/path/to/your/extension/mysql-test
   ```

## 故障排除

### 扩展无法加载

检查错误日志并验证 VEB 内容：
```bash
make show_veb
tail -f /var/log/mysql/error.log
```

### 函数未找到

验证安装和注册：
```sql
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;
```

### 构建错误

```bash
# Verify mysql_config is available
which mysql_config
mysql_config --version

# Check compiler version
gcc --version  # or clang --version

# Verify CMake version (3.18+ required)
cmake --version
```

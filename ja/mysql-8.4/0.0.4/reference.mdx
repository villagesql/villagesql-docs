標準SQLインターフェイスを使用して、拡張機能メタデータおよびサーバー状態を照会します。

---

## システムビュー

### INFORMATION_SCHEMA.EXTENSIONS

現在インストールされているすべてのVillageSQL拡張機能を一覧表示します。

<Note>
`INSTALL EXTENSION`および`UNINSTALL EXTENSION`はVillageSQL SQL拡張機能です。
これらは標準MySQL 8.4構文の一部ではありません。
</Note>

**既知のカラム:**

| カラム | 型 | 説明 |
|--------|------|-------------|
| `EXTENSION_NAME` | varchar | インストールされた拡張機能の名前 |
| `EXTENSION_VERSION` | varchar | 拡張機能が報告したバージョン文字列 |

**例:**
```sql
-- Install an extension (VillageSQL-specific syntax)
INSTALL EXTENSION vsql_complex;

-- List all installed extensions
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;

-- Check a specific extension's version
SELECT EXTENSION_VERSION
FROM INFORMATION_SCHEMA.EXTENSIONS
WHERE EXTENSION_NAME = 'vsql_complex';
```

**例示出力**（実際のバージョン文字列はインストールされた拡張機能に依存）:
```
+------------------+-------------------+
| EXTENSION_NAME   | EXTENSION_VERSION |
+------------------+-------------------+
| vsql_complex     | 0.0.1             |
| vsql_uuid        | 0.2.1             |
+------------------+-------------------+
```

`EXTENSION_NAME`の値は小文字で、`make_extension()`に渡された名前と一致します。

このビューは現在インストールされている状態を反映しています。

### INFORMATION_SCHEMA.COLUMNS (カスタム型)

カスタム拡張機能型を使用するカラムは、標準の`INFORMATION_SCHEMA.COLUMNS`ビューから表示されます。カスタム型は`extension_name.type_name`として`DATA_TYPE`および`COLUMN_TYPE`カラムに表示されます（例: `vsql_complex.COMPLEX`）。

**例:**
```sql
-- Find all columns using custom extension types
SELECT TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE DATA_TYPE LIKE '%.%'
ORDER BY TABLE_SCHEMA, TABLE_NAME;

-- Find columns using a specific extension's types
SELECT TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE DATA_TYPE LIKE 'vsql_complex.%';
```

**サンプル出力:**
```
+--------------+------------+-------------+---------------------+
| TABLE_SCHEMA | TABLE_NAME | COLUMN_NAME | DATA_TYPE           |
+--------------+------------+-------------+---------------------+
| mydb         | signals    | impedance   | vsql_complex.COMPLEX|
| mydb         | signals    | frequency   | vsql_complex.COMPLEX|
+--------------+------------+-------------+---------------------+
```

### INFORMATION_SCHEMA.EXTENSION_REGISTRATION

読み込まれた各拡張機能のメモリ内VEF登録構造体をJSONドキュメントとして公開します。`INSTALL EXTENSION`後にサーバーが拡張機能の関数、型、システム変数を正しく解析したかどうかを検証するために使用します。

```sql
SELECT EXTENSION_NAME, NEGOTIATED_PROTOCOL, REGISTRATION_JSON
FROM INFORMATION_SCHEMA.EXTENSION_REGISTRATION
WHERE EXTENSION_NAME = 'vsql_complex';
```

| カラム | 型 | 説明 |
|--------|------|-------------|
| `EXTENSION_NAME` | `VARCHAR(64)` | インストールされた拡張機能の名前。 |
| `NEGOTIATED_PROTOCOL` | `BIGINT UNSIGNED` | 拡張機能とサーバー間で交渉されたVEFプロトコルバージョン。 |
| `REGISTRATION_JSON` | `longtext` | `vef_registration_t`構造体のJSONシリアライズ（`funcs`および`types`配列を含む）。 |

## 一般的なクエリ

### 拡張機能の依存関係を検索

アンインストールする前に、特定の拡張機能の型を使用するカラムを検索します:

```sql
SELECT TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE DATA_TYPE LIKE 'vsql_complex.%';
```

### すべての拡張機能とそのカスタム型カラムを一覧表示

```sql
-- All installed extensions
SELECT EXTENSION_NAME, EXTENSION_VERSION
FROM INFORMATION_SCHEMA.EXTENSIONS
ORDER BY EXTENSION_NAME;

-- All columns using custom types across all extensions
SELECT TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE DATA_TYPE LIKE '%.%'
ORDER BY DATA_TYPE, TABLE_SCHEMA, TABLE_NAME;
```

### 拡張機能型を使用するテーブルを検索

```sql
SELECT TABLE_SCHEMA, TABLE_NAME, COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE DATA_TYPE LIKE 'vsql_complex.%'
ORDER BY TABLE_SCHEMA, TABLE_NAME;
```

## システム変数

### veb_dir

実行時読み取り専用。サーバーが`.veb`拡張機能バンドルファイルを検索するディレクトリのパス。`my.cnf`の`[mysqld]`で設定され、サーバー再起動なしでは変更できません。

```sql
SHOW VARIABLES LIKE 'veb_dir';
```

**スコープ:** グローバル、実行時読み取り専用。`my.cnf`で構成します:

```ini
[mysqld]
veb_dir=/path/to/extensions/
```

単一のディレクトリのみをサポートします。配置およびトラブルシューティングについては[拡張機能の管理](/mysql-8.4/0.0.4/managing)を参照してください。

---

### villagesql_server_version

実行時読み取り専用のグローバル変数。サーバーバイナリにコンパイルされたVillageSQLバージョン文字列を返します。これは`villagesql_schema_version`とは異なり、内部メタデータカタログバージョンを追跡するものではありません。

```sql
SELECT @@villagesql_server_version;
-- Example output: 0.0.4 (release builds)
-- Development builds include a commit suffix: 0.0.4-dev-03d249dd

-- Show all VillageSQL system variables at once
SHOW VARIABLES LIKE 'villagesql_%';
```

**スコープ:** グローバル、読み取り専用。実行時に設定できません。

---

### villagesql_vef_server_protocol

実行時読み取り専用のグローバル変数。このサーバービルドがサポートする最高のVEFプロトコルバージョンを返します。拡張機能開発者は、この値を`types.h`のプロトコル定数と比較し、登録時に利用可能なABI機能を判断できます。

```sql
SELECT @@villagesql_vef_server_protocol;
```

| プロパティ | 値 |
|----------|-------|
| **スコープ** | グローバル |
| **アクセス** | 読み取り専用 |
| **型** | 無符号整数（`0`–`255`） |
| **現在の値** | `4`（`VEF_PROTOCOL_4`） |

この値はコンパイル時定数`vef_server_protocol_version`を反映しており、実行時に変更できません。

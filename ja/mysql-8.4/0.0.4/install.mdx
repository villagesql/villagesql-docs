title: "拡張機能のインストール"
description: "拡張機能を使用して VillageSQL に新しい機能を追加する方法。"
---

VillageSQL の拡張機能をインストールして、カスタム型、関数、および機能をデータベースに追加します。

## コマンド構文

```sql
INSTALL EXTENSION extension_name;
```

<Info>
  拡張機能は、コンパイルされたライブラリとメタデータを含む `.veb`（VillageSQL 拡張機能バンドル）ファイルとして配布されます。
</Info>

### 拡張機能の命名規則

VillageSQL では、異なる文脈で異なる命名規則を使用します：

- **SQL コマンド:** アンダースコアを使用します: `INSTALL EXTENSION vsql_uuid`
- **リポジトリ名:** ハイフンを使用します: `github.com/villagesql/vsql-uuid`
- **ファイル名:** アンダースコアを使用します: `vsql_uuid.veb`
- **manifest.json:** SQL と一致させるためにアンダースコアを使用します: `"name": "vsql_uuid"`

**例:**
```bash
# Clone from GitHub repo (hyphens in URL)
git clone https://github.com/villagesql/vsql-uuid

# But .veb file uses underscores
ls vsql_uuid.veb

# Install with underscores (no quotes)
INSTALL EXTENSION vsql_uuid;
```

## 前提条件

- 実行中の VillageSQL Server インスタンス
- 管理者アクセス（root または同等）

## 組み込み拡張機能のインストール

VillageSQL に含まれる組み込み拡張機能は既に veb_dir にあります。単に有効にするだけです：

```sql
-- Connect to VillageSQL
mysql -u root -p

-- Install the extension
INSTALL EXTENSION vsql_complex;
```

### インストールの確認

```sql
-- List installed extensions
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;
```

**出力:**
```
+------------------+-------------------+
| EXTENSION_NAME   | EXTENSION_VERSION |
+------------------+-------------------+
| vsql_complex     | 0.0.1             |
+------------------+-------------------+
```

### 機能のテスト

```sql
-- Create a database first
CREATE DATABASE test_db;
USE test_db;

-- Test extension functions
CREATE TABLE test (id INT, value COMPLEX);
INSERT INTO test VALUES (1, '(3,4)');
SELECT complex_abs(value) FROM test;  -- Returns 5.0

-- Clean up
DROP TABLE test;
DROP DATABASE test_db;
```

## 外部拡張機能のインストール

別途ダウンロードまたはビルドした拡張機能の場合:

<Info>
  外部拡張機能をインストールする前に、サーバーで `veb_dir` が設定されている必要があります。[veb_dir の設定](/mysql-8.4/0.0.4/managing#configuring-veb_dir)を参照してください。
</Info>

### 1. .veb ファイルをコピー

サーバーの拡張機能ディレクトリを見つけ、`.veb` ファイルをそのディレクトリにコピーします：

```sql
-- Find the extension directory
SHOW VARIABLES LIKE 'veb_dir';
```

```bash
cp /path/to/my_extension.veb /path/to/veb_dir/
```

### 2. 拡張機能のインストール

```sql
INSTALL EXTENSION my_extension;
```

### 3. インストールの確認

```sql
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS WHERE EXTENSION_NAME = 'my_extension';
```

## トラブルシューティング

| 問題 | 解決策 |
|-------|-------|
| `Extension not found` | `.veb` ファイルが `veb_dir` に存在することを確認してください: `SHOW VARIABLES LIKE 'veb_dir'` |
| `Permission denied` | ファイル権限を確認してください: `chmod 644 extension.veb` |
| `extension name mismatch` | 拡張機能の内部名が `.veb` ファイル名と一致していません。拡張機能を再ビルドしてください。 |
| `vef_register not found` | `.veb` ファイルは有効な VEF エントリーポイントをエクスポートしていません。正しい SDK で再ビルドしてください。 |
| `vef_register returned an error: ...` | 拡張機能の登録に失敗しました。付加されたメッセージを参照してください。 |

詳細なトラブルシューティングについては、[拡張機能の管理](/mysql-8.4/0.0.4/managing)を参照してください。

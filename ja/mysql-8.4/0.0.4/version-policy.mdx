## Overview

VillageSQLは、デプロイメントおよびアップグレードに関する意思決定をサポートする明確なバージョン管理および互換性ポリシーを採用しています。

## VillageSQL バージョン管理

VillageSQLは、各リリースにおける変更の性質と影響を伝えるために、[Semantic Versioning](https://semver.org/)（SemVer）を使用しています。

### バージョン形式

バージョンは以下の形式に従います：**MAJOR.MINOR.PATCH**

```
0.0.1
│ │ │
│ │ └─ PATCH version
│ └─── MINOR version
└───── MAJOR version
```

### バージョン増分ルール

- **MAJORバージョン**（例：1.0.0 → 2.0.0）：コードの修正やデータベースのマイグレーションを必要とする破壊的変更
- **MINORバージョン**（例：0.1.0 → 0.2.0）：後方互換性を保ちながら追加された新機能および機能
- **PATCHバージョン**（例：0.0.1 → 0.0.2）：後方互換性を保ちながら行われたバグ修正およびマイナーな改善

<Note>
  VillageSQLは現在、1.0未満の開発フェーズ（バージョン0.x.x）にあります。この段階では、プラットフォームの安定化に伴い、APIおよび拡張インターフェイスがより頻繁に変更される可能性があります。
</Note>

### 開発ビルド

プレリリースサフィックス（例：`0.0.4-dev`）を含むバージョンは開発ビルドです。デフォルトでは、開発ビルドで初期化されたデータベースからのアップグレードをサーバーがブロックします。開発ビルドはアップグレード互換性のテストが行われておらず、破壊的なスキーマまたはプロトコルの変更を含む可能性があります。

これを許可するには、`--villagesql-allow-unsafe-dev-upgrade`フラグを指定してサーバーを起動します：

```bash
mysqld --villagesql-allow-unsafe-dev-upgrade --datadir=/path/to/data
```

フラグを指定しない場合、サーバーは開発バージョンのデータベースを検出した時点でエラーで終了します：

```
Upgrading from a development version (0.0.4-dev) is not allowed. Use --villagesql-allow-unsafe-dev-upgrade to permit this.
```

フラグを指定すると、サーバーは警告をログに記録し、処理を続行します：

```
Upgrading from a development version (0.0.4-dev); --villagesql-allow-unsafe-dev-upgrade was specified.
```

フラグが指定されているが現在のバージョンが開発ビルドでない場合、サーバーはエラーで終了します：

```
--villagesql-allow-unsafe-dev-upgrade specified, but current version (0.0.3) is not a development version.
```

または、スキーマアップグレードが実行されていない場合：

```
--villagesql-allow-unsafe-dev-upgrade specified, but no upgrade is being performed.
```

## 拡張機能の互換性

VillageSQLの拡張機能はバージョン固有です。あるバージョンのVillageSQL用に構築された拡張機能は、別のバージョンでは動作しない可能性があります。

VillageSQLをアップグレードする際には：

1. 新バージョンの拡張機能互換性を確認する
2. 必要に応じて拡張機能を更新または再構築する
3. 本番環境へのデプロイ前に開発環境で拡張機能をテストする

詳細なアップグレード手順については、[アップグレードガイド](/guides/upgrade)を参照してください。

## 0.0.3 からの SDK 変更

**プロトコル2の拡張機能は再構築が必要です。** VEFプロトコル3はv0.0.4時点で安定しており、サーバーはロード時にプロトコル2の拡張機能を拒否します。プロトコル3 APIを使用し、型安全なラッパー（`StringArg`、`IntArg`、`StringResult`など）で再構築してください。

**`vef_version_t::extra`型が`const char*`に修正されました。** フィールドは0.0.3では`unsigned char*`でした。ほとんどの拡張機能は直接アクセスしません。SDKが`manifest.json`バージョンフィールドから自動的に値を設定します。直接アクセスする必要がある場合は、0.0.4 SDKでコンパイルする際、`(unsigned char *)`キャストを削除してください。

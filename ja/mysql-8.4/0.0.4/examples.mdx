`vsql_complex` 拡張機能は、VEF SDK を使用したカスタム型の VillageSQL の参照実装です—本番環境で使用可能な拡張機能パターンを示しています。

**ソース:** [VillageSQL リポジトリ](https://github.com/villagesql/villagesql-server/tree/main/villagesql/examples/vsql-complex) にある `villagesql/examples/vsql-complex/`

---

## vsql_complex が提供する機能

複素数 (a + bi) 用の COMPLEX 型で、算術演算、ユーティリティ、集計をサポートします。

**使用例:**
```sql
INSTALL EXTENSION vsql_complex;

CREATE TABLE signals (
    id INT PRIMARY KEY,
    impedance COMPLEX
);

INSERT INTO signals VALUES (1, '(50.0,10.0)');

SELECT
    complex_real(impedance) as resistance,
    complex_imag(impedance) as reactance,
    complex_abs(impedance) as magnitude
FROM signals;
```

## VEF 登録パターン

**ファイル: `src/complex.cc`**

vsql_complex は VEF SDK と `VEF_GENERATE_ENTRY_POINTS()` を使用しています:

```cpp
#include <villagesql/vsql.h>

// Type name constants as char arrays — required as non-type template parameters
// (NTTPs) so that make_type can auto-generate VDF names like "COMPLEX::from_string".
static constexpr const char kComplexTypeName[] = "COMPLEX";
static constexpr const char kComplex2TypeName[] = "COMPLEX2";

// Type objects: encode/decode/compare VDFs are embedded via template parameters.
// No separate .func() registration is needed for type operations.
constexpr auto COMPLEX =
    vsql::make_type<kComplexTypeName>()
        .persisted_length(kComplexSize)
        .max_decode_buffer_length(64)
        .from_string<&complex_from_string>()
        .to_string<&complex_to_string>()
        .compare<&complex_compare>()
        .intrinsic_default_str("(0,0)")
        .build();

constexpr auto COMPLEX2 =
    vsql::make_type<kComplex2TypeName>()
        .persisted_length(kComplexSize)
        .max_decode_buffer_length(64)
        .from_string<&complex2_from_string>()
        .to_string<&complex_to_string>()
        .compare<&complex_compare>()
        .hash<&complex2_hash>()
        .intrinsic_default_str("(0,0)")
        .build();

using namespace vsql;

VEF_GENERATE_ENTRY_POINTS(
    make_extension()
        .type(COMPLEX)
        .type(COMPLEX2)
        // Arithmetic operations
        .func(make_func<&complex_add_impl>("complex_add")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .param(COMPLEX)
                  .deterministic()
                  .build())
        .func(make_func<&complex_subtract_impl>("complex_subtract")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .param(COMPLEX)
                  .deterministic()
                  .build())
        .func(make_func<&complex_multiply_impl>("complex_multiply")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .param(COMPLEX)
                  .deterministic()
                  .build())
        .func(make_func<&complex_divide_impl>("complex_divide")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .param(COMPLEX)
                  .deterministic()
                  .build())
        // Utility functions
        .func(make_func<&complex_real_impl>("complex_real")
                  .returns(REAL)
                  .param(COMPLEX)
                  .build())
        .func(make_func<&complex_imag_impl>("complex_imag")
                  .returns(REAL)
                  .param(COMPLEX)
                  .build())
        .func(make_func<&complex_abs_impl>("complex_abs")
                  .returns(REAL)
                  .param(COMPLEX)
                  .build())
        .func(make_func<&complex_conjugate_impl>("complex_conjugate")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .build())
        // Aggregate functions
        .func(make_aggregate_func<ComplexSumState, &complex_sum_result>(
                  "complex_sum")
                  .returns(COMPLEX)
                  .param(COMPLEX)
                  .clear<&complex_sum_clear>()
                  .accumulate<&complex_sum_accumulate>()
                  .build()))
```

**主要パターン:**
- 1 つのマクロ呼び出しですべて登録 — 手動 SQL は不要
- 型オブジェクトは `constexpr` 変数で、`vsql::make_type<kName>()` で構築され、`kName` は非型テンプレートパラメータとして使用される `static constexpr const char[]` です
- 型操作 (`.from_string<>()`, `.to_string<>()`, `.compare<>()`, `.hash<>()`) は型オブジェクトに埋め込まれており、それらのための別途の `.func()` 呼び出しは不要です
- `.compare()` で ORDER BY およびインデックスが有効化されます；`.hash()` はオプションです
- `.intrinsic_default_str("(0,0)")` は、NOT NULL 列に NULL を INSERT IGNORE または UPDATE IGNORE で割り当てた際のデフォルト値を設定します
- 関数登録には `make_func<&impl>("name")` と `.build()` を使用します
- 集計登録には `make_aggregate_func<State, &result_fn>("name")` と `.clear<&fn>()`、`.accumulate<&fn>()` を使用します；結果関数のシグネチャは `void(const State&, ResultWrapper)` です

## バイナリストレージ形式

COMPLEX は **16 バイト** (リトルエンディアン) で格納されます:
- バイト 0-7: 実部 (double)
- バイト 8-15: 虚部 (double)

**エンコード/デコード (complex.cc):**
```cpp
bool encode_complex(unsigned char *buffer, size_t buffer_size,
                    const char *from, size_t from_len, size_t *length);
bool decode_complex(const unsigned char *buffer, size_t buffer_size,
                    char *to, size_t to_buffer_size, size_t *to_length);
```

プラットフォーム非依存のバイトオーダー関数を使用してクロスプラットフォーム互換性を確保しています。

## ラッパーファンクション

VEF SDK は型付き引数および結果のラッパーを提供するため、実装は C++ 型を使用して Raw プロトコル構造体ではなく動作します:

**算術例 (complex.cc から):**

```cpp
void complex_add_impl(CustomArg in_l, CustomArg in_r, CustomResult out) {
  // Handle NULL inputs and validate arguments
  // ...
  store_complex(out.buffer().data(), Complex{lhs.re + rhs.re, lhs.im + rhs.im});
  out.set_length(kComplexSize);
}
```

**ユーティリティ関数ラッパー (complex.cc から):**

```cpp
void complex_real_impl(CustomArg in, RealResult out) {
  // Handle NULL and validate
  // ...
  out.set(cx.re);
}

void complex_abs_impl(CustomArg in, RealResult out) {
  // Handle NULL and validate
  // ...
  out.set(sqrt(cx.re * cx.re + cx.im * cx.im));
}
```

## キー実装パターン

| パターン | 使用例 |
|---------|-----|
| **固定長ストレージ** | 型ビルダーで `.persisted_length()` を設定 |
| **プラットフォーム非依存** | double 用にカスタムバイトオーダー関数を使用 |
| **NULL ハンドリング** | 型付き引数ラッパーで `in.is_null()` を呼び出し |
| **エラーハンドリング** | 結果ラッパーで `out.error("message")` を呼び出し |
| **結果出力** | 結果ラッパーで `out.set(value)` / `out.set_length(n)` を呼び出し |

## カスタム型でのデータのエクスポート/インポート

VillageSQL はカスタム型で `SELECT INTO OUTFILE` および `LOAD DATA INFILE` をサポートし、カスタム型の値を保持したままデータをエクスポート/インポートできます。

### SELECT INTO OUTFILE

カスタム型はエクスポート時に文字列表現にシリアライズされます:

```sql
INSTALL EXTENSION vsql_complex;

-- Create table with custom type
CREATE TABLE signals (
    id INT PRIMARY KEY,
    reading COMPLEX
);

INSERT INTO signals VALUES
    (1, '(3.0,4.0)'),
    (2, '(5.0,12.0)'),
    (3, '(-1.0,2.0)');

-- Export to file
SELECT * FROM signals INTO OUTFILE '/tmp/signals_export.txt';
```

**ファイル内容 (`/tmp/signals_export.txt`):**
```
1	(3.00,4.00)
2	(5.00,12.00)
3	(-1.00,2.00)
```

### LOAD DATA INFILE

エクスポートしたデータをテーブルに戻します:

```sql
-- Create new table with same schema
CREATE TABLE signals_imported (
    id INT PRIMARY KEY,
    reading COMPLEX
);

-- Import data
LOAD DATA INFILE '/tmp/signals_export.txt' INTO TABLE signals_imported;

-- Verify
SELECT * FROM signals_imported;
```

### カスタム区切り文字でのエクスポート

カスタムフィールドおよび行終端文字を使用できます:

```sql
SELECT * FROM signals
INTO OUTFILE '/tmp/signals_csv.txt'
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```

**出力:**
```
"1","(3.00,4.00)"
"2","(5.00,12.00)"
"3","(-1.00,2.00)"
```

### VDF 関数でのエクスポート

拡張機能関数を使用して計算値をエクスポートします:

```sql
SELECT
    id,
    reading,
    complex_abs(reading) AS magnitude,
    complex_real(reading) AS real_part,
    complex_imag(reading) AS imag_part
INTO OUTFILE '/tmp/signals_computed.txt'
FROM signals;
```

<Note>
  カスタム型は文字列表現形式でエクスポートされます。バイナリエクスポートは `SELECT INTO DUMPFILE` で現在サポートされていません。
</Note>

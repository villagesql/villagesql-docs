VillageSQL Server インスタンスを起動し、接続して、拡張システムを試してみましょう。

## ステップ 1: VillageSQL のインストール

### オプション A: Docker (推奨)

ホスト側にインストールすることなく、コンテナで VillageSQL を実行します。

```bash
docker run -d -e MYSQL_ALLOW_EMPTY_PASSWORD=yes -p 3306:3306 villagesql/server:stable
```

### オプション B: シェルスクリプト

公式のインストールスクリプトを使用して、マシンに VillageSQL を直接インストールします。これにより、サーバーバイナリがダウンロードされ、プラットフォーム用に構成されます。

```bash
curl https://villagesql.com/shell | sh
```

スクリプトを実行する前に確認するには: `curl https://villagesql.com/shell | less`

### オプション C: ソースコードからのビルド

開発またはカスタムビルドの場合は、[ソースコードからのクローンとビルドガイド](/mysql-8.4/0.0.4/source)に従って、最新のコードからコンパイルします。

## ステップ 2: サーバーへの接続

サーバーが起動したら、任意の標準の MySQL クライアントを使用して接続します。

```bash
mysql -u root -p
```

## ステップ 3: 最初の拡張機能のインストール

VillageSQL には、すぐにインストールできるサンプル拡張機能が付属しています。

`vsql_complex` 拡張機能をインストールして、複素数のサポートを追加します。

```sql
INSTALL EXTENSION vsql_complex;
```

インストールを確認します。

```sql
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;
```

`vsql_complex` がリストされているはずです。

詳細については、[拡張機能のインストール](/mysql-8.4/0.0.4/install) を参照してください。

## ステップ 4: 拡張データ型の使用

拡張機能がアクティブになったので、テーブルで `COMPLEX` データ型をネイティブ型と同様に使用できます。

```sql
-- Create a database and use it
CREATE DATABASE demo;
USE demo;

-- Create a table using the new COMPLEX type
CREATE TABLE electronic_components (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    impedance COMPLEX
);

-- Insert complex number values (using string literals)
INSERT INTO electronic_components VALUES
    (1, 'Resistor', '(100,0)'),
    (2, 'Inductor', '(0,50)'),
    (3, 'Capacitor', '(25,-75)');

-- Query the data
SELECT * FROM electronic_components;

-- Use extension functions
SELECT
    name,
    impedance,
    complex_abs(impedance) AS magnitude,
    complex_real(impedance) AS resistance,
    complex_imag(impedance) AS reactance
FROM electronic_components;
```

拡張機能をアンインストールするには:

```sql
UNINSTALL EXTENSION vsql_complex;
```

## トラブルシューティング

### サーバーが起動しない

一般的な問題:
- ポート 3306 がすでに使用されている: サーバーが別のポートを使用するように構成します。
- 権限: ファイルが読み取り/実行可能であることを確認します。

このガイドでは、GDB デバッグ、サーバーのテストスイートの実行、VillageSQL への貢献について説明します。このガイドを使用する前に、[ソースからビルド](/mysql-8.4/0.0.4/source)で動作するビルドを取得してください。

## GDB によるデバッグ

開発およびトラブルシューティングのため、VillageSQL を GDB で実行できます。

**Linux:**
```bash
gdb --args $HOME/build/villagesql/bin/mysqld --gdb --datadir=$HOME/mysql-data/data --basedir=$HOME/build/villagesql
```

**macOS:**
```bash
gdb --args ~/build/villagesql/bin/mysqld --gdb --datadir=~/mysql-data/data --basedir=~/build/villagesql
```

**GDB 内:**
```
(gdb) run                    # Start the server
(gdb) break function_name    # Set breakpoints
(gdb) continue              # Resume execution
(gdb) bt                    # Show backtrace on crash
```

**MySQL デバッグ用の一般的な GDB コマンド:**
```
# Break on specific SQL command execution
(gdb) break mysql_execute_command

# Break on extension loading
(gdb) break Sql_cmd_install_extension::execute

# Print variable values
(gdb) print variable_name

# Step through code
(gdb) step      # Step into functions
(gdb) next      # Step over functions
```

## テストの実行

VillageSQL には、機能を検証するためのユニットテストおよびリグレッションテストが含まれています。

### ユニットテスト

VillageSQL 固有のユニットテストを実行する（ビルドディレクトリ内で）:

```bash
make -j10 villagesql-unit-tests && ctest -L villagesql
```

またはすべてのユニットテストを実行する:

```bash
ctest --output-on-failure
```

### リグレッションテスト

mysql-test ディレクトリに移動します:
```bash
cd mysql-test
```

**フル MySQL テストスイートを実行（遅い）:**
```bash
./mysql-test-run.pl --parallel=auto
```

**VillageSQL 固有のテストのみを実行:**
```bash
./mysql-test-run.pl --suite=villagesql
```

**すべてのサブスイートを含む VillageSQL テストを実行:**
```bash
./mysql-test-run.pl --do-suite=villagesql --parallel=auto
```

<Note>
  **--suite と --do-suite の違い:**
  - `--suite=villagesql` はトップレベルの villagesql テストのみを実行します
  - `--do-suite=villagesql` はサブスイート（insert、select、stored_procedure など）を含むすべてのテストを実行します
</Note>

### 個別テストの実行

テスト名で特定のテストを実行します:

```bash
./mysql-test-run.pl villagesql.complex_index
```

サブスイートからテストを実行します:

```bash
./mysql-test-run.pl --suite=villagesql/insert
```

### テスト結果の更新

機能を変更し、期待されるテスト出力を更新する必要がある場合:

```bash
./mysql-test-run.pl --record villagesql.complex_index
```

<Warning>
  **テスト記録:** `.result` ファイルへの変更を常にレビューし、バグではなく意図した動作を反映していることを確認してください。`--record` フラグは期待される出力を盲目的に上書きします。
</Warning>

### テストの移植性

テスト出力にテストランナーの一時ディレクトリからのパスが含まれる場合、`.test` ファイル内に以下のディレクティブを追加して、他のマシンで動作しない絶対パスを記録結果に含めないようにします:

```sql
--replace_result $MYSQLTEST_VARDIR MYSQLTEST_VARDIR
```

### テスト失敗のデバッグ

VillageSQL 固有のログメッセージ（`LogVSQL()` で出力）はデフォルトで抑制されています。テスト実行中にエラーログに表示させるには:

```bash
./mysql-test-run.pl --mysqld=--log-error-verbosity=3 villagesql.complex_index
```

完全なエラーログは `mysql-test/var/log/mysqld.1.err` にあります。

拡張機能のリグレッションテストの作成については、[開発ガイド](/mysql-8.4/0.0.4/development)を参照してください。

## バイナリのインストール

VillageSQL をシステム全体にインストールするには（適切な権限が必要です）:

```bash
cmake --build build --target install
```

これはデフォルトでバイナリを `/usr/local/mysql/` にインストールします。インストールプレフィックスをカスタマイズすることもできます:

```bash
cmake -S . -B build -DCMAKE_INSTALL_PREFIX=/opt/villagesql
cmake --build build --target install
```

## 貢献方法

VillageSQL に貢献する場合は、コーディング規約および開発ワークフローについて、[貢献ガイドライン](https://github.com/villagesql/villagesql-server/blob/main/CONTRIBUTING.md)を参照してください。

---

VillageSQLはMySQLのトラッキングフォークであり、MySQLのドロップインリプレースメントです。MySQLのプラグインおよびコンポーネントインターフェースはVillageSQLでもMySQLと同様に動作します。VillageSQLはVillageSQL Extension Framework (VEF)を提供しています。新しい機能を追加する場合（カスタムSQL関数、カスタムタイプ、統合など）には、VEF拡張機能が適切なモデルです。

## どこに何を配置するか

新しいものを構築する際は、自身のリポジトリでVEF拡張機能を構築してください。プラグインやコンポーネントも動作しますが、VEFが適切なモデルです — [拡張機能をプラグインやコンポーネントの代わりに使用する理由](#why-extensions-instead-of-plugins-and-components)を参照してください。

VillageSQL Serverリポジトリに属する貢献:

| | |
|---|---|
| VEF自体の改善（新しいフックタイプ、SDK機能） | [serverリポジトリ](https://github.com/villagesql/villagesql-server)へのPR |
| サーバーのバグ修正 | [serverリポジトリ](https://github.com/villagesql/villagesql-server)でのイシューまたはPR |

## 拡張機能をプラグインやコンポーネントの代わりに使用する理由

MySQLのプラグインおよびコンポーネントはVillageSQLで動作します。VEF拡張機能にはさらに以下の利点があります:

- **カスタムタイプ** — 新しいカラムタイプを定義可能（プラグインおよびUDFでは不可能）
- **パッケージング** — `.veb`ファイルとして配布され、`INSTALL EXTENSION`でインストール
- **再起動不要** — 拡張機能は実行時にロードおよびアンロード可能
- **バージョン管理** — 拡張機能マニフェストで宣言され、サーバーによって強制

## 新しい拡張機能の開始

[vsql-extension-template](https://github.com/villagesql/vsql-extension-template)をクローンまたはフォークしてください。これは動作するHello World拡張機能、マニフェスト形式、および完全なCMakeビルド設定を含んでいます。

リポジトリを取得したら、[拡張機能の作成](/mysql-8.4/0.0.4/create)がVEF SDKを説明しています。

## VEFがサポートしていない場合

VEFはまだ成長中です。まだサポートされていないフックや機能が必要な場合:

- **イシューを提出** — [serverリポジトリ](https://github.com/villagesql/villagesql-server/issues)で必要な機能を説明
- **既存のイシューをアップボート** — 要望が既に存在する場合
- **PRを提出** — VEFにフックを追加 — フレームワークの改善はserverリポジトリで歓迎されます

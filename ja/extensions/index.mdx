VillageSQL には、組み込みの拡張機能が付属しており、VillageSQL チームによって構築および保守される公式拡張機能もサポートされています。`INSTALL EXTENSION` を使用して、これらの拡張機能のいずれかをインストールできます。

## 組み込み拡張機能

<Warning>
  ソースコードからビルドする場合、`vsql_complex` と `vsql_simple` のみが含まれます。以下の公式拡張機能は、個別のリポジトリで保守されています。
</Warning>

### vsql_complex

**ステータス:** VillageSQL に組み込み
**リポジトリ:** [villagesql-server/villagesql/examples/vsql-complex](https://github.com/villagesql/villagesql-server/tree/main/villagesql/examples/vsql-complex)

電気工学、信号処理、および科学計算のための完全な算術演算をサポートする複素数データ型。

**機能:**
- 複素数 (a + bi) を格納するための `COMPLEX` 列型
- 文字列リテラル形式: `'(real,imaginary)'` (例: `'(3.0,4.0)'`)
- 算術演算: 加算、減算、乗算、除算
- ユーティリティ: `complex_real()`、`complex_imag()`、`complex_abs()`、`complex_conjugate()`
- バイナリストレージ: 16 バイト (2 つの double 型)

**インストール:**
```sql
INSTALL EXTENSION vsql_complex;
```

**例:**
```sql
CREATE TABLE signals (
    id INT PRIMARY KEY,
    impedance COMPLEX,
    frequency_response COMPLEX
);

INSERT INTO signals VALUES
    (1, '(50.0,0.0)', '(0.95,0.31)');

SELECT
    complex_abs(impedance) as magnitude,
    complex_add(impedance, frequency_response) as total
FROM signals;
```

## コア拡張機能

以下の拡張機能は、個別のリポジトリで保守されており、個別にビルドおよびインストールする必要があります。それぞれに独自のインストール手順とドキュメントが用意されています。

### vsql_ai

**リポジトリ:** [villagesql/vsql-ai](https://github.com/villagesql/vsql-ai)

AI を活用した SQL 関数。SQL クエリのための AI プロンプトおよび埋め込み関数。Anthropic Claude、Google Gemini、OpenAI、およびローカルの Ollama をサポートします。

### vsql_crypto

**リポジトリ:** [villagesql/vsql-crypto](https://github.com/villagesql/vsql-crypto)

暗号化関数。ハッシュ (MD5、SHA-1、SHA-256、SHA-512 など)、HMAC、AES 暗号化、パスワードハッシュ、および OpenSSL を使用したセキュアなランダムデータの生成。

### vsql_cube

**リポジトリ:** [villagesql/vsql-cube](https://github.com/villagesql/vsql-cube)

N 次元ボックスおよびポイント型。最大 100 次元にわたって、幾何学的ボックスとポイントを格納およびクエリします。コンストラクタ、アクセサ、述語、距離、および幾何関数が利用可能です。VillageSQL 0.0.4 が必要です。

### vsql_http

**リポジトリ:** [villagesql/vsql-http](https://github.com/villagesql/vsql-http)

HTTP クライアント関数。SQL クエリから HTTP リクエスト (GET、POST、PUT、DELETE、PATCH) を送信し、JSON 形式のレスポンスを受け取ります。また、URL のエンコードとデコードもサポートします。

### vsql_network_address

**リポジトリ:** [villagesql/vsql-network-address](https://github.com/villagesql/vsql-network-address)

ネットワークアドレス型。IPv4、IPv6、および MAC アドレスデータ型。検証、比較、およびネットワーク操作をサポートします。

### vsql_uuid

**リポジトリ:** [villagesql/vsql-uuid](https://github.com/villagesql/vsql-uuid)

UUID データ型および関数。SQL でネイティブに UUID を生成および操作します。v1、v3、v4、v5、v6、および v7 をサポートします。ネイティブの 16 バイト UUID 型、タイムスタンプ抽出、および比較が可能です。

## コミュニティ拡張機能

コミュニティ拡張機能は、VillageSQL 拡張フレームワーク (VEF) を使用して、サードパーティによって構築されています。VillageSQL チームによって保守されるものではありません。

### vsql_statistics

**リポジトリ:** [ronaldbradford/vsql-statistics](https://github.com/ronaldbradford/vsql-statistics)
**メンテナー:** [Ronald Bradford](https://github.com/ronaldbradford)

データ分析のための統計集計関数。IQR、四分位数、中央値、および Tukey のフェンスを使用した外れ値検出 — `STATS_IQR`、`STATS_Q1`、`STATS_Q3`、`STATS_MEDIAN`、`STATS_IQR_LOWER_FENCE`、`STATS_IQR_UPPER_FENCE`。VillageSQL 0.0.4 が必要です。

### prometheus_exporter

**リポジトリ:** [ProxySQL/vsql-prometheus-exporter](https://github.com/ProxySQL/vsql-prometheus-exporter)
**メンテナー:** [ProxySQL](https://github.com/ProxySQL)

組み込みの Prometheus メトリクスエクスポートツール。MySQL サーバーのメトリクスを、Prometheus 形式の HTTP `/metrics` エンドポイントで公開します。グローバルステータス、グローバル変数、InnoDB メトリクス、およびレプリケーションステータスが含まれます。

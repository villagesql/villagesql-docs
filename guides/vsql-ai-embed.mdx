`create_embed` calls an LLM embedding provider directly from within the database engine and returns a JSON array of floats. Using it inside a `BEFORE INSERT` trigger means every new row gets its embedding atomically — if the embedding call fails, the insert is rolled back.

## Installation

```sql
INSTALL EXTENSION vsql_ai;
```

Verify installation:

```sql
SELECT * FROM information_schema.EXTENSIONS WHERE EXTENSION_NAME = 'vsql_ai';
```

## Function Signature

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `provider` | `VARCHAR` | `'google'` or `'openai'` |
| `model` | `VARCHAR` | Embedding model identifier |
| `api_key` | `VARCHAR` | Your API key for the provider |
| `text` | `TEXT` | The source text to embed |
| **Returns** | **`VARCHAR`** | JSON array of floats (e.g., `'[0.012, -0.034, ...]'`) |

<Note>
  `create_embed` supports `'google'` and `'openai'` only. Anthropic does not offer an embeddings API. Use `ai_prompt` for Anthropic inference.
</Note>

**Supported embedding models:**
- Google: `'gemini-embedding-001'`, `'gemini-embedding-2-preview'`
- OpenAI: `'text-embedding-3-small'`, `'text-embedding-3-large'`

## Implementation: Automated Embedding via Trigger

The following example creates a trigger that generates and stores an embedding whenever a row is inserted into the `knowledge_base` table.

<Note>
  Session variables like `@embed_api_key` are connection-scoped and do not persist between sessions. The trigger reads `@embed_api_key` from the session that issues the `INSERT`. If the variable is not set in that session, `create_embed` receives `NULL` and the insert fails. Set the variable at the start of each connection before inserting rows, or pass the key via your application's connection initialization.
</Note>

```sql
-- Store the API key in a session variable
SET @embed_api_key = 'your-openai-api-key';

-- Define the table with a JSON column for embedding storage
CREATE TABLE knowledge_base (
    id INT AUTO_INCREMENT PRIMARY KEY,
    document_chunk TEXT NOT NULL,
    embedding JSON,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Create the trigger for automated embedding
DELIMITER //
CREATE TRIGGER tr_generate_embedding
BEFORE INSERT ON knowledge_base
FOR EACH ROW
BEGIN
    SET NEW.embedding = CAST(create_embed(
        'openai',
        'text-embedding-3-small',
        @embed_api_key,
        NEW.document_chunk
    ) AS CHAR);
END //
DELIMITER ;

-- Test the trigger
SET @embed_api_key = 'your-openai-api-key';
INSERT INTO knowledge_base (document_chunk)
VALUES ('VillageSQL implements custom types natively on top of MySQL 8.4 LTS.');

-- Verify the result
SELECT id, LEFT(document_chunk, 40) AS snippet, JSON_LENGTH(embedding) AS dimensions
FROM knowledge_base;
```

## Comparison: VillageSQL vs. Standard MySQL 8.4 + App Layer

| Feature | Standard MySQL 8.4 + App Layer | VillageSQL `create_embed` |
| :--- | :--- | :--- |
| **Atomic Consistency** | Risk of partial success (DB write succeeds, API call fails). | Trigger failure rolls back the entire row insertion. |
| **Implementation** | Python/Node.js middleware required. | Pure SQL; works within existing ETL pipelines. |
| **Concurrency** | Managed by the application connection pool. | Managed by VillageSQL's internal HTTP worker pool. |

## Frequently Asked Questions

#### What happens if the embedding API is unavailable during an insert?
The trigger raises an error and the `INSERT` is rolled back. The row is not written to the table. Every row in the table is guaranteed to have a valid embedding.

#### Should the embedding column be NOT NULL?
It depends on your consistency requirements. `NOT NULL` guarantees every row has an embedding but makes the table unavailable if the API is down. A nullable column allows inserts to succeed and lets you backfill embeddings separately.

#### How do I choose the right embedding model?
The dimension count depends on the model. `text-embedding-3-small` outputs 1536 floats by default. Check your provider's documentation for the correct value and size your storage column accordingly.

#### Can I update embeddings when the source text changes?
Yes. Add a `BEFORE UPDATE` trigger with the same `SET NEW.embedding = create_embed(...)` logic.

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `FUNCTION create_embed does not exist` | Extension not installed | Run `INSTALL EXTENSION vsql_ai` |
| Insert fails with API error | Provider unreachable or key invalid | Test the function directly: `SELECT create_embed('openai', 'text-embedding-3-small', @key, 'test')` to see the raw error |
| Trigger not firing | Trigger created on wrong table or schema | Verify: `SHOW TRIGGERS LIKE 'knowledge_base'` |
| Returns `NULL` | Null argument or unsupported provider | Check that all four arguments are non-null |

VillageSQL provides native `INET` and `CIDR` data types for storing IPv4 and IPv6 addresses and network prefixes. The types enforce format validity at insert time, index efficiently as fixed-width binary, and display in human-readable notation without a conversion step.

## Installation

```sql
INSTALL EXTENSION vsql_network_address;
```

Verify installation:

```sql
SELECT * FROM information_schema.EXTENSIONS WHERE EXTENSION_NAME = 'vsql_network_address';
```

## Type Reference

| Type | Description | Example |
| :--- | :--- | :--- |
| `INET` | A single IPv4 or IPv6 host address | `'192.168.1.1'`, `'2001:db8::1'` |
| `CIDR` | A network prefix with mask length | `'10.0.0.0/8'`, `'2001:db8::/32'` |

## Function Reference

| Function | Returns | Description |
| :--- | :--- | :--- |
| `inet_text(inet)` | `VARCHAR` | Text representation of an INET value with mask |
| `inet_abbrev(inet)` | `VARCHAR` | Abbreviated notation (e.g., `'10.0.0.1/32'` â†’ `'10.0.0.1'`) |
| `inet_family(inet)` | `INT` | `4` for IPv4, `6` for IPv6 |
| `inet_masklen(inet)` | `INT` | Prefix length of an INET value (e.g., `24` for `/24`) |
| `cidr_abbrev(cidr)` | `VARCHAR` | Abbreviated CIDR notation (e.g., `'10.0.0.0/8'` â†’ `'10/8'`) |

| `macaddr_trunc(macaddr)` | `MACADDR` | OUI prefix (first 3 bytes) |

## Storing and Querying Network Addresses

```sql
-- Create a table using native INET and CIDR types
CREATE TABLE access_control (
    id INT AUTO_INCREMENT PRIMARY KEY,
    trusted_range CIDR NOT NULL,
    description VARCHAR(255),
    INDEX idx_network (trusted_range)
);

-- Insert mixed IPv4 and IPv6 CIDR blocks
INSERT INTO access_control (trusted_range, description) VALUES
    ('10.0.0.0/8',        'Internal LAN'),
    ('192.168.0.0/16',    'Office Network'),
    ('2001:db8:a::/48',   'Production IPv6 Segment');

-- Display network information
SELECT
    description,
    trusted_range,
    cidr_abbrev(trusted_range) AS abbreviated
FROM access_control;

-- Store individual host addresses with family metadata
CREATE TABLE host_inventory (
    id INT AUTO_INCREMENT PRIMARY KEY,
    hostname VARCHAR(255),
    ip_address INET NOT NULL
);

INSERT INTO host_inventory (hostname, ip_address) VALUES
    ('web-01', '10.0.1.10'),
    ('db-01',  '10.0.2.5'),
    ('gw-v6',  '2001:db8::1');

SELECT hostname, ip_address, inet_family(ip_address) AS ip_version
FROM host_inventory;
```

## Comparison: VillageSQL vs. Standard MySQL 8.4

| Feature | Standard MySQL 8.4 | VillageSQL 0.x |
| :--- | :--- | :--- |
| **Data Storage** | `VARBINARY(16)` (opaque bytes) | `INET` / `CIDR` (typed, validated) |
| **Input Validation** | Application-side; invalid addresses stored silently | Rejected at insert time with a descriptive error |
| **IPv6 Handling** | Requires manual byte-level handling | Native 128-bit typed storage |
| **Indexability** | Functional index on `VARBINARY` | Direct B-Tree index on `CIDR` or `INET` column |
| **Display** | Binary output requires `INET6_NTOA()` | Human-readable output without conversion |

## Frequently Asked Questions

#### Can I store both IPv4 and IPv6 in the same INET column?
Yes. The `INET` type accepts both IPv4 and IPv6 addresses in the same column. VillageSQL stores them in a normalized internal representation.

#### What happens if I insert an invalid IP address?
VillageSQL raises an error and rejects the insert. The `INET` and `CIDR` types enforce format validity at the database level.

#### Can I index a CIDR column?
Yes. A standard B-Tree index on a `CIDR` column works for equality lookups. Because CIDR values are stored as fixed-width binary, the index is compact.

#### How do I get the family (IPv4 vs IPv6) of a stored address?
Use `inet_family(column)`: returns `4` for IPv4 and `6` for IPv6.

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `Extension 'vsql_network_address' is already installed` | Extension was already loaded | The extension is installed and available. No action needed. |
| Insert rejected with format error | Invalid IP or CIDR notation | Verify the address is a valid IPv4/IPv6 string with a valid prefix length |
| Functions not available | Extension not installed | Run `INSTALL EXTENSION vsql_network_address` |
| Index not used on `CIDR` column | Query uses a function on the indexed column | Avoid wrapping the indexed column in functions in the `WHERE` clause |

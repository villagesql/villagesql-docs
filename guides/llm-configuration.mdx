VillageSQL's AI functions — `ai_prompt` and `create_embed` — take the API key as a direct function argument. The recommended pattern is to set the key once per session as a user variable and reference that variable in queries, so the raw key string never appears in your SQL text.

## Installation

```sql
INSTALL EXTENSION vsql_ai;
```

Verify installation:

```sql
SELECT * FROM information_schema.EXTENSIONS WHERE EXTENSION_NAME = 'vsql_ai';
```

## Supported Providers

| Provider | Identifier | Functions |
| :--- | :--- | :--- |
| Anthropic (Claude) | `'anthropic'` | `ai_prompt` |
| Google (Gemini) | `'google'` | `ai_prompt`, `create_embed` |
| OpenAI (GPT) | `'openai'` | `ai_prompt`, `create_embed` |

## API Key Pattern

```sql
-- Set the key once at the start of a session
SET @anthropic_key = 'your-anthropic-api-key';
SET @openai_key    = 'your-openai-api-key';
SET @google_key    = 'your-google-api-key';

-- Reference the variable in queries — the key is never visible in the SQL text
SELECT ai_prompt(
    'anthropic',
    'claude-sonnet-4-6',
    @anthropic_key,
    CONCAT('Summarize: ', content)
) AS summary
FROM articles
WHERE summary IS NULL
LIMIT 5;
```

## Provider Examples

### Anthropic (Claude)

```sql
SET @api_key = 'your-anthropic-api-key';

SELECT ai_prompt(
    'anthropic',
    'claude-sonnet-4-6',
    @api_key,
    'What is the capital of France?'
) AS response;
```

**Available Claude models:**
- `claude-sonnet-4-6` — recommended for complex tasks and coding
- `claude-haiku-4-5-20251001` — fastest response time, lowest cost
- `claude-opus-4-6` — highest capability; slower and more expensive than Sonnet

### Google (Gemini)

```sql
SET @api_key = 'your-google-api-key';

SELECT ai_prompt(
    'google',
    'gemini-2.5-flash',
    @api_key,
    'What is the capital of France?'
) AS response;

-- Embeddings with Google
SELECT create_embed(
    'google',
    'gemini-embedding-001',
    @api_key,
    'VillageSQL is a MySQL 8.4 extension platform.'
) AS embedding_json;
```

### OpenAI

```sql
SET @api_key = 'your-openai-api-key';

SELECT ai_prompt(
    'openai',
    'gpt-4o-mini',
    @api_key,
    'What is the capital of France?'
) AS response;

-- Embeddings with OpenAI
SELECT create_embed(
    'openai',
    'text-embedding-3-small',
    @api_key,
    'VillageSQL is a MySQL 8.4 extension platform.'
) AS embedding_json;
```

## Frequently Asked Questions

#### Is the API key visible in query logs?
User variables (`@api_key`) expand at execution time. The raw key may appear in the general query log if it's enabled. Disable general logging in production or use a log masking tool to redact sensitive values.

#### Can I use different providers for different queries in the same session?
Yes. Set separate variables for each provider and reference the appropriate one in each query:
```sql
SET @anthropic_key = '...';
SET @openai_key = '...';
-- Use anthropic for one query, openai for another
```

#### How do I rotate an API key?
Run `SET @api_key = 'new-key'` in the current session. For application-managed connections, update the key in your application configuration and reconnect.

#### Are local LLM providers like Ollama or vLLM supported?
Not currently. VillageSQL's AI functions support `'anthropic'`, `'google'`, and `'openai'` as providers.

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `FUNCTION ai_prompt does not exist` | Extension not installed | Run `INSTALL EXTENSION vsql_ai` |
| Returns `NULL` with no error | API key is null or empty | Confirm `SELECT @api_key` returns the expected value |
| `Access denied` or auth error | Wrong or expired key | Test the key against the provider's API directly |
| Wrong provider name | Typo or unsupported provider | Use exactly `'anthropic'`, `'google'`, or `'openai'` |

`INSTALL EXTENSION` deploys `.veb` (VillageSQL Extension Bundle) files, which package compiled C++ shared objects together with their SQL schema definitions. A single command handles binary registration and schema initialization atomically, so the extension is either fully installed or not at all.

## Installation

The extension management engine is native to VillageSQL. Confirm the VEB directory path before placing extension files:

```sql
-- Verify the VEB load path
SELECT @@veb_dir;
```

## Command Syntax

| Parameter | Type | Description | Required |
| :--- | :--- | :--- | :--- |
| `extension_name` | Identifier | Name of the extension, matching the `.veb` filename without the extension. | Yes |

**Install:**
```sql
INSTALL EXTENSION extension_name;
```

**Uninstall:**
```sql
UNINSTALL EXTENSION extension_name;
```

## Installing, Verifying, and Removing an Extension

The following example walks through installing, verifying, and removing an extension.

```sql
-- 1. Verify the VEB directory path
SELECT @@veb_dir;

-- 2. Install an extension from the configured veb_dir
INSTALL EXTENSION vsql_crypto;

-- 3. Verify installation status
SELECT * FROM information_schema.EXTENSIONS
WHERE EXTENSION_NAME = 'vsql_crypto';

-- 4. Remove the extension and its associated metadata
UNINSTALL EXTENSION vsql_crypto;
```

## Comparison: VillageSQL vs. Standard MySQL 8.4

| Feature | Standard MySQL 8.4 | VillageSQL 0.x (.veb) |
| :--- | :--- | :--- |
| **Atomic Deployment** | Manual `.so` install + separate SQL scripts. | Single command handles binaries and schema together. |
| **Dependency Resolution** | No native dependency checking. | Manifest-based checks for required system libraries. |
| **Version Tracking** | System variables only (manual). | Native semantic versioning in `information_schema.EXTENSIONS`. |
| **Package Format** | Loose `.so` or `.dll` files. | Encapsulated `.veb` (binary + SQL + metadata). |

## Frequently Asked Questions

#### What is a .veb file?
A VillageSQL Extension Bundle (`.veb`) is an archive containing the compiled shared object (`.so`/`.dll`), SQL schema migration files, and a manifest describing the extension's version and dependencies.

#### Can I install an extension without restarting the server?
Yes. `INSTALL EXTENSION` loads the shared object into the running server without requiring a restart.

#### What happens if installation fails partway through?
The operation is atomic. If any step fails — binary load, schema migration, or dependency check — the server rolls back to the pre-installation state.

#### How do I check which extensions are installed?
```sql
SELECT * FROM information_schema.EXTENSIONS;
```

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `Can't open shared library` | `.veb` file not in `@@veb_dir` | Verify `SELECT @@veb_dir` and copy the file there |
| `Extension already exists` | Extension already loaded | Run `UNINSTALL EXTENSION name` before reinstalling |
| `Dependency not satisfied` | Required library missing from the host | Install the dependency listed in the error message |

VillageSQL's `ai_prompt` and `create_embed` functions call cloud AI providers directly from within the database engine. The API key travels with each function call, so you can use different providers for different queries in the same session.

<Note>
  This guide covers which providers are supported and what each one does. For how to manage API keys safely across sessions, see [Managing AI Provider API Keys](/guides/llm-configuration).
</Note>

## Installation

```sql
INSTALL EXTENSION vsql_ai;
```

Verify installation:

```sql
SELECT * FROM information_schema.EXTENSIONS WHERE EXTENSION_NAME = 'vsql_ai';
```

## Supported Providers

| Provider | Identifier | `ai_prompt` | `create_embed` |
| :--- | :--- | :--- | :--- |
| Anthropic (Claude) | `'anthropic'` | ✓ | — |
| Google (Gemini) | `'google'` | ✓ | ✓ |
| OpenAI (GPT) | `'openai'` | ✓ | ✓ |

## SQL Examples for Each Provider

### Using Anthropic (Claude) for inference

```sql
SET @api_key = 'your-anthropic-api-key';

SELECT
    ticket_id,
    ai_prompt(
        'anthropic',
        'claude-sonnet-4-6',
        @api_key,
        CONCAT('Summarize this support ticket in 10 words: ', description)
    ) AS summary
FROM support_tickets
WHERE status = 'open'
LIMIT 5;
```

### Using Google (Gemini) for inference and embeddings

```sql
SET @api_key = 'your-google-api-key';

-- Inference
SELECT ai_prompt(
    'google',
    'gemini-2.5-flash',
    @api_key,
    'List 3 benefits of in-database AI inference.'
) AS response;

-- Embeddings
SELECT create_embed(
    'google',
    'gemini-embedding-001',
    @api_key,
    'VillageSQL adds native AI functions to MySQL 8.4.'
) AS embedding_json;
```

### Using OpenAI (GPT) for inference and embeddings

```sql
SET @api_key = 'your-openai-api-key';

-- Inference
SELECT ai_prompt(
    'openai',
    'gpt-4o-mini',
    @api_key,
    'List 3 benefits of in-database AI inference.'
) AS response;

-- Embeddings
SELECT create_embed(
    'openai',
    'text-embedding-3-small',
    @api_key,
    'VillageSQL adds native AI functions to MySQL 8.4.'
) AS embedding_json;
```

## Comparison: VillageSQL vs. Standard MySQL 8.4

| Feature | VillageSQL 0.x | Standard MySQL 8.4 |
| :--- | :--- | :--- |
| **Cloud AI Support** | Built-in `ai_prompt` and `create_embed` functions | None; requires external application logic |
| **Data Movement** | DB calls the provider directly; data stays in the query pipeline | Data must egress to application layer |
| **Integration** | Direct SQL call; no middleware | Requires Python/Node.js to bridge DB and LLM |

## Frequently Asked Questions

#### Are local providers like Ollama or vLLM supported?
Not currently. `ai_prompt` and `create_embed` support `'anthropic'`, `'google'`, and `'openai'` only.

#### Can I use different providers in the same query?
Yes. Call `ai_prompt` or `create_embed` with different provider and key values in the same `SELECT`. Each function call is independent.

#### Does the model need to be specified on every call?
Yes. The model identifier is the second argument to `ai_prompt` and `create_embed`. There's no server-level default model setting.

#### Can I use ai_prompt inside a trigger?
Yes. It's a scalar function and can be called from `BEFORE INSERT` or `BEFORE UPDATE` triggers. If the LLM call fails, the triggering DML statement also fails.

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `FUNCTION ai_prompt does not exist` | Extension not installed | Run `INSTALL EXTENSION vsql_ai` |
| `Unknown provider` error | Unsupported provider name | Use exactly `'anthropic'`, `'google'`, or `'openai'` |
| Returns `NULL` | Null argument | Check that all four arguments are non-null |
| Query times out | Model taking too long to respond | Reduce prompt complexity; use a faster model variant |

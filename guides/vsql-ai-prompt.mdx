`ai_prompt` is a scalar function that executes LLM inference directly within the VillageSQL execution engine. Pass your provider, model, API key, and prompt — the function handles the HTTP call and returns the model's response as a plain string.

## Installation

```sql
INSTALL EXTENSION vsql_ai;
```

Verify installation:

```sql
SELECT * FROM information_schema.EXTENSIONS WHERE EXTENSION_NAME = 'vsql_ai';
```

## Function Signature

| Parameter | Data Type | Description |
| :--- | :--- | :--- |
| `provider` | `VARCHAR` | `'anthropic'`, `'google'`, or `'openai'` |
| `model` | `VARCHAR` | Model identifier (e.g., `'claude-sonnet-4-6'`) |
| `api_key` | `VARCHAR` | Your API key for the provider |
| `prompt` | `TEXT` | The input text — typically a column value or `CONCAT` expression |
| **Returns** | **`VARCHAR`** | The model's response as a plain string |

## Classifying and Tagging Data with ai_prompt

Classifying support ticket text using Claude:

```sql
-- Store the API key in a session variable (never hardcode in queries)
SET @api_key = 'your-anthropic-api-key';

SELECT
    ticket_id,
    ai_prompt(
        'anthropic',
        'claude-sonnet-4-6',
        @api_key,
        CONCAT(
            'Categorize this support ticket into exactly one of: ',
            'Billing, Technical, Feature Request. ',
            'Return only the category name.\n\nTicket: ',
            customer_feedback
        )
    ) AS category
FROM support_tickets
WHERE category IS NULL
LIMIT 10;
```

Batch-updating a column with AI-generated content:

```sql
SET @api_key = 'your-openai-api-key';

UPDATE products
SET tags = ai_prompt(
    'openai',
    'gpt-4o-mini',
    @api_key,
    CONCAT('Generate 3 comma-separated SEO tags for: ', product_description)
)
WHERE tags IS NULL;
```

## Comparison: VillageSQL vs. Standard MySQL 8.4

| Feature | VillageSQL (`vsql_ai`) | Standard MySQL 8.4 |
| :--- | :--- | :--- |
| **In-Database Inference** | Native via `ai_prompt`. | Not available; requires external UDFs or ETL. |
| **Workflow** | SQL-native; works with `JOIN`, `WHERE`, and `UPDATE`. | Requires Python/Node.js glue code. |
| **Concurrency** | Managed by MySQL's internal thread pool. | Subject to external application connection limits. |

## Frequently Asked Questions

#### Which providers does ai_prompt support?
`ai_prompt` supports `'anthropic'` (Claude), `'google'` (Gemini), and `'openai'` (GPT). Pass the provider name as the first argument.

#### Does ai_prompt support streaming?
No. `ai_prompt` is a synchronous scalar function that returns the complete response. This makes it suitable for use inside `SELECT`, `UPDATE`, and `VIEW` definitions.

#### How should I manage API keys?
Store the key in your application configuration and pass it as a session variable at the start of each session: `SET @api_key = '...'`. Don't embed key literals directly in views or stored procedures where they appear in query logs.

#### Can I use ai_prompt inside a trigger?
Yes. Because it's a scalar function, it can be called from `BEFORE INSERT` or `BEFORE UPDATE` triggers. If the LLM call fails, the triggering DML statement also fails.

## Troubleshooting

| Problem | Cause | Solution |
| :--- | :--- | :--- |
| `FUNCTION ai_prompt does not exist` | Extension not installed | Run `INSTALL EXTENSION vsql_ai` |
| Returns `NULL` | Null input or unsupported provider | Check that all four arguments are non-null; verify the provider name is `'anthropic'`, `'google'`, or `'openai'` |
| `Access denied for API key` | Key not valid or expired | Test with a known-good key; check the provider's key status |
| Query times out | LLM provider latency | Increase `wait_timeout`; reduce the dataset with `LIMIT` |

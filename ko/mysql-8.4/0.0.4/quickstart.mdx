VillageSQL Server 인스턴스를 실행하고, 연결한 후 확장 시스템을 사용해 보세요.

## 단계 1: VillageSQL 설치

### 옵션 A: Docker (권장)

호스트 설치 없이 컨테이너에서 VillageSQL을 실행합니다:

```bash
docker run -d -e MYSQL_ALLOW_EMPTY_PASSWORD=yes -p 3306:3306 villagesql/server:stable
```

### 옵션 B: 쉘 스크립트

공식 설치 스크립트를 사용하여 직접 머신에 VillageSQL을 설치합니다. 해당 스크립트는 플랫폼에 맞는 서버 바이너리를 다운로드하고 구성합니다.

```bash
curl https://villagesql.com/shell | sh
```

스크립트를 실행하기 전에 확인하려면: `curl https://villagesql.com/shell | less`

### 옵션 C: 소스에서 빌드

개발 또는 맞춤 빌드를 위해 [소스에서 복제 및 빌드 가이드](/mysql-8.4/0.0.4/source)를 따르세요.

## 단계 2: 서버에 연결

서버가 실행된 후, 표준 MySQL 클라이언트로 연결합니다:

```bash
mysql -u root -p
```

## 단계 3: 첫 번째 확장 설치

VillageSQL은 즉시 설치 가능한 예제 확장이 포함되어 있습니다.

복소수 지원을 추가하려면 `vsql_complex` 확장을 설치합니다:

```sql
INSTALL EXTENSION vsql_complex;
```

설치 확인:

```sql
SELECT * FROM INFORMATION_SCHEMA.EXTENSIONS;
```

`vsql_complex`가 목록에 표시되어야 합니다.

자세한 내용은 [확장 설치](/mysql-8.4/0.0.4/install)를 참조하세요.

## 단계 4: 확장된 데이터 타입 사용

확장이 활성화되면, 테이블에서 `COMPLEX` 데이터 타입을 네이티브 타입처럼 사용할 수 있습니다.

```sql
-- Create a database and use it
CREATE DATABASE demo;
USE demo;

-- Create a table using the new COMPLEX type
CREATE TABLE electronic_components (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    impedance COMPLEX
);

-- Insert complex number values (using string literals)
INSERT INTO electronic_components VALUES
    (1, 'Resistor', '(100,0)'),
    (2, 'Inductor', '(0,50)'),
    (3, 'Capacitor', '(25,-75)');

-- Query the data
SELECT * FROM electronic_components;

-- Use extension functions
SELECT
    name,
    impedance,
    complex_abs(impedance) AS magnitude,
    complex_real(impedance) AS resistance,
    complex_imag(impedance) AS reactance
FROM electronic_components;
```

확장 제거 방법:

```sql
UNINSTALL EXTENSION vsql_complex;
```

## 문제 해결

### 서버 시작 안됨

흔한 문제:
- 포트 3306이 이미 사용 중: 서버를 다른 포트로 구성하세요
- 권한: 파일이 읽기/실행 가능하도록 확인하세요
